## Task 3: Create `/init-docs` Skill

Claude Code skill definition with four modes.

**Files:**
- Create: `skill/init-docs/SKILL.md`

- [ ] **Step 1: Write skill definition**

Create `skill/init-docs/SKILL.md` with YAML front matter matching existing skill pattern (see `skill/stark-review-plan/SKILL.md` for format):

```yaml
---
name: init-docs
description: >
  Scaffold dev docs structure into any repo. Modes: --template (empty skeleton),
  --backfill (generate from git history), --upgrade (migrate existing docs),
  --clean (remove skeleton). Use when the user says "init docs", "setup docs",
  "scaffold docs", or invokes /init-docs.
---
```

Skill body must define:
- **Arguments:** `--template`, `--backfill`, `--upgrade`, `--clean` (combinable: `--upgrade --backfill`)
- **Constants:** paths to templates at `~/.claude/code-review/standards/templates/`
- **No-arg mode:** show the four options with one-line descriptions, ask which mode
- **`--template` mode:** create `docs/{specs,plans,adr,guides,reference,architecture}/`, copy templates (ADR template → `docs/adr/0000-template.md`, mkdocs.yml → root with repo name substituted, PR template → `.github/pull_request_template.md`, staleness config → `.doc-staleness.yml`, index → `docs/index.md`). Add CODEOWNERS if missing. Idempotent — skip existing files/dirs.
- **`--backfill` mode:** run `--template` first. Then: read `git log --oneline --all` (last 200 commits), read `gh pr list --state merged --limit 50 --json number,title,body,mergedAt`, analyze codebase (package.json, requirements.txt, Makefile, CI configs, Dockerfile). Generate ADRs for technology choices (conservative — language, framework, DB, major libs only). Generate stub specs from significant merged PRs. Create guides from scripts/CI/Makefile patterns. Wire into mkdocs.yml nav. Commit all.
- **`--upgrade` mode:** scan repo for existing Markdown docs outside `docs/`. Classify each as spec, ADR, guide, reference, or architecture. Move via `git mv` into standard layout. Update internal Markdown links. If mkdocs.yml exists, restructure nav. Run `--template` to fill gaps.
- **`--clean` mode:** prompt for confirmation. Remove only skeleton files (templates, empty dirs, mkdocs.yml, .doc-staleness.yml). Do NOT delete user-generated content in `docs/` subdirs.

```python
import re

def extract_spec_link(pr_body: str | None) -> str | None:
    """Extract spec link from PR description. Returns path/URL, 'N/A', or None."""
    if not pr_body:
        return None
    match = re.search(r'##\s*Spec:\s*(.+)', pr_body)
    if not match:
        return None
    value = match.group(1).strip()
    if not value or value.startswith('<!--'):
        return None
    return value

def resolve_spec_content(spec_link: str, cwd: str) -> str | None:
    """Read spec file content. Returns content string or None if unresolvable."""
    if spec_link == "N/A":
        return None
    if spec_link.startswith("http"):
        # URL — can't resolve locally, agent will need to fetch
        return None
    spec_path = os.path.join(cwd, spec_link)
    if os.path.isfile(spec_path):
        with open(spec_path, 'r') as f:
            return f.read()
    return None
```

- [ ] **Step 2: Write the GitHub Action workflow**

Create `standards/workflows/doc-staleness.yml`:

```yaml
name: Doc Staleness Check
on:
  pull_request:
    types: [opened, synchronize]

jobs:
  check-staleness:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
        with:
          fetch-depth: 0  # Full history for git log dates

      - name: Check for stale docs
        uses: actions/github-script@v7
        with:
          script: |
            const fs = require('fs');
            const path = require('path');
            const { execSync } = require('child_process');

            // Load config
            let config = { threshold_days: 90, exclude_paths: ['docs/adr/', 'docs/specs/', 'docs/plans/'] };
            try {
              const configFile = fs.readFileSync('.doc-staleness.yml', 'utf8');
              // Simple YAML parse for our flat structure
              const lines = configFile.split('\n');
              for (const line of lines) {
                const match = line.match(/^threshold_days:\s*(\d+)/);
                if (match) config.threshold_days = parseInt(match[1]);
              }
              // Parse exclude_paths
              const excludes = [];
              let inExcludes = false;
              for (const line of lines) {
                if (line.match(/^exclude_paths:/)) { inExcludes = true; continue; }
                if (inExcludes && line.match(/^\s+-\s+(.+)/)) {
                  excludes.push(line.match(/^\s+-\s+(.+)/)[1].trim());
                } else if (inExcludes && !line.match(/^\s/)) {
                  inExcludes = false;
                }
              }
              if (excludes.length) config.exclude_paths = excludes;
            } catch (e) { /* use defaults */ }

            const threshold = Date.now() - (config.threshold_days * 24 * 60 * 60 * 1000);
            const stale = [];

            // Find all markdown files in docs/
            function walkDir(dir) {
              if (!fs.existsSync(dir)) return [];
              const files = [];
              for (const entry of fs.readdirSync(dir, { withFileTypes: true })) {
                const full = path.join(dir, entry.name);
                if (entry.isDirectory()) files.push(...walkDir(full));
                else if (entry.name.endsWith('.md')) files.push(full);
              }
              return files;
            }

            const docs = walkDir('docs');
            for (const file of docs) {
              // Check exclusions
              if (config.exclude_paths.some(ex => file.startsWith(ex))) continue;

              try {
                const ts = execSync(`git log -1 --format=%ct -- "${file}"`, { encoding: 'utf8' }).trim();
                if (!ts) continue;
                const lastModified = parseInt(ts) * 1000;
                if (lastModified < threshold) {
                  const date = new Date(lastModified).toISOString().split('T')[0];
                  stale.push(`- \`${file}\` (last updated ${date})`);
                }
              } catch (e) { /* skip */ }
            }

            if (stale.length > 0) {
              const body = `**Stale docs detected** (not updated in ${config.threshold_days}+ days):\n\n${stale.join('\n')}\n\nThese may be outdated. Consider reviewing them if your changes affect related areas.`;
              await github.rest.issues.createComment({
                owner: context.repo.owner,
                repo: context.repo.repo,
                issue_number: context.issue.number,
                body: body
              });
            }
```

Note: The `github-script` action's `script` field is a multi-line YAML string (indicated by `|`). The YAML is structurally complete — the `script` block ends at the dedent back to the `steps` level. The hand-rolled YAML parsing for `.doc-staleness.yml` is intentionally simple and works for the flat config structure. If more complex config is needed later, replace with `js-yaml` npm package.
