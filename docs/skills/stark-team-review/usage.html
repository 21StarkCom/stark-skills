stark-review

Multi-agent PR code review — 3 LLMs × 6 domains with autonomous fix loop
USAGE GUIDE
Quick Start
# Review current branch's PR /stark-review
# Review a specific PR /stark-review 91
# Dry run — review only, no fixes, no GitHub posting /stark-review 91 --dry-run
# Limit fix rounds /stark-review 91 --rounds 2

Operating Modes
Full Mode
Review-Only

Full Mode

Review + autonomous fix loop + build/test validation. Activated when test_command is configured and PR is not from a fork. Commits fixes and pushes to PR branch.

Review-Only Mode

Single review pass, no fixes applied. Triggered by: fork PRs, missing test_command config, or --dry-run flag. Findings posted as comments only.

What You Get

PR Comments

Per-agent raw findings posted under stark-claude, stark-codex, and stark-gemini bot identities. Plus a unified orchestrator summary with classifications.

Automated Fixes

Code fixes committed and pushed directly to the PR branch (full mode). Each round commits with message fix: address review findings (round N).

Bug Issues

Critical/high severity unresolved findings become GitHub Issues with stark-review label. Max 5 per run, deduplicated against existing issues.

Review History

Persisted to ~/.claude/code-review/history/{org}/{repo}/{pr}/ — summary.md, rounds.json (with timing), and prompt-assessment.md.

Noise Analysis

Misalignment root causes identified with concrete improvement actions: CLAUDE.md additions, config changes, prompt tuning targets.

Common Workflows

Standard PR Review

Push your branch, then run /stark-review. The skill auto-detects the PR, dispatches 18 sub-agents, fixes what it can, and posts the summary to GitHub.

Pre-Merge Quality Gate

Run /stark-review --rounds 1 for a single review pass. Check the summary for critical/high findings before merging. Combine with /stark-pr-flow for the full workflow.

Dry Run for Local Feedback

Use /stark-review --dry-run to get review feedback without posting to GitHub or making fixes. Good for checking code quality before opening a PR.
