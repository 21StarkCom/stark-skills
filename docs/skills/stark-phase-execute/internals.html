Internal Architecture — Data Flow & Auth

Auth Split (Critical)

USER PAT unset GH_TOKEN — PR creation, merge, issue close. Shows Aryeh as actor.

BOT TOKEN export GH_TOKEN=$(github_app.py token) — review comment posting, Project V2 mutations. Shows stark-claude[bot].

Rule: Always unset before PR/merge ops. Only export for review posting & project fields. Never cross the streams.

Subagent Architecture

Implementation: Agent tool (foreground) per task. Prompt includes issue body, branch name, repo root. Subagent commits but never pushes.

Review: multi_review.py dispatches 18 sub-agents (3 LLMs × 6 domains) via ThreadPoolExecutor. Returns JSON. Round loop managed by THIS skill, not multi_review.py.

Fix: Optional subagent for complex fixes inside worktree.

Worktree Isolation

Reviews happen in /tmp/review-{REPO}-pr{N}, not the main working tree. This prevents review/fix operations from contaminating the orchestrator's git state.

Lifecycle: create → fetch PR head → review → fix → push → remove worktree → delete temp branch.

Crash recovery: If worktree exists from a previous crash, reuse it.

Config Cascade

Hierarchy: repo .code-review/config.json → org .code-review/config.json → ~/.claude/code-review/config.json

Key fields: test_command, deploy_command, review domains, agent preambles

Project V2: .github/project-config.json enables project-based task fetching, spec validation, and status transitions.

Error Handling Philosophy

Never block the phase. Every failure is logged and the next task starts. Merge failures escalate to --admin. CI failures are bypassed (flagged). Branch cleanup happens in the error handler.

Rate limits: wait 60s, retry once, then log and move on.
Subagent timeout: skip task, continue.

Observability

JSON log: ~/.claude/code-review/history/{ORG}/{REPO}/phase-{SLUG}-{ts}.json
Incrementally updated per task. Includes per-task timing, findings by severity/agent, fix rates.

Timestamped stdout: [HH:MM:SS] prefixed logs
5-min checkpoints: printed at phase transitions after 5min elapsed
TodoWrite tasks: parent + child per issue, one in_progress at a time

Key Scripts & Dependencies

multi_review.py

PR review orchestrator. Runs 18 sub-agents (3 LLMs × 6 domains) via ThreadPoolExecutor. No --rounds flag — it runs one round only. Returns JSON findings. --dry-run prevents GitHub posting.

github_app.py

Multi-app GitHub auth. Supports stark-claude, stark-codex, stark-gemini. Token generation, PR review posting (pr_review()), caching. Private keys from macOS Keychain.

github_projects.py

GitHub Projects V2 integration. get_items(), transition_status(), set_field(), check_spec_completeness(). Used when .github/project-config.json exists.

Skill Delegates

/stark-plan-to-tasks — auto-decompose when no issues exist
/stark-release — version bump, tag, GitHub Release
Both invoked via Skill tool.

CLI Dependencies

gh — GitHub CLI (PRs, issues, merge)
claude, codex, gemini — AI CLIs (sub-agents)
git — worktrees, branches, push

Security Invariants

LLM output never interpolated in shell (temp files only). GH_TOKEN split enforced per operation. --dangerouslySkipPermissions required for full autonomy.
