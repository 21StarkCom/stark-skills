Architectural Highlights

1. Total Autonomy

This skill explicitly overrides all user-confirmation gates in all sub-workflows. There are no "wait for approval" or "are you sure?" prompts. It forces CI bypasses if needed via --admin merge and delegates to /stark-release which natively respects autonomous invocation.

2. Worktree Isolation

Review loops occur in a detached worktree (/tmp/review-{REPO}-pr{N}) rather than corrupting the primary repository state. The skill calculates merge bases, runs multi_review.py, applies fixes, and commits back to the origin PR branch cleanly.

3. Auto-Decomposition Fallback

If executed before issues exist, the skill will actively scan docs/ for the target PLAN_FILE, execute /stark-plan-to-tasks dynamically, and then re-fetch the newly generated GitHub issues to begin the task loop.

4. Dual Auth Strategy

Uses explicit token swaps: export GH_TOKEN=... (bot token via Python auth script) for GH Project manipulations and posting review comments, but resets to unset GH_TOKEN (User PAT via CLI) for issue reading, PR creation, and merges.
