Auth Token Architecture

User PAT (gh native auth)

Used for: creating PRs, merging PRs, creating/closing issues, reading issues, merge operations.

Activated by: unset GH_TOKEN — lets gh fall back to native auth.

Shows as: Aryeh (the user), not a bot.

Bot Token (stark-claude[bot])

Used for: posting review comments, GitHub Projects V2 mutations (status transitions, field updates).

Activated by: export GH_TOKEN=$($PYTHON github_app.py --app stark-claude token)

Key source: macOS Keychain (STARK_CLAUDE_PRIVATE_KEY, base64 PEM). Cached 1hr at ~/.cache/github-app-token.json.

Critical Rule

Never create PRs or merge with bot token. Never post reviews with user PAT. The auth split is enforced by explicit unset GH_TOKEN / export GH_TOKEN=... around each operation. Failure to unset causes PRs to appear as bot, breaking attribution.

Task Fetching Strategy

Project-Based (Preferred)

Reads .github/project-config.json → queries GitHub Projects V2 API for items with Status=Ready for Agent and AI Suitability ∈ {Autonomous, Assisted}. Uses github_projects.get_items() via bot token. Falls back to label-based if no config or no items.

Label-Based (Fallback)

Queries gh api /repos/{ORG_REPO}/issues?labels=plan:{SLUG}&state=open via user PAT. Filters out phase tracking issues (body starts with task checklist, no ## What section). Extracts sp:N, risk:*, type:* from labels.

Auto-Decompose (Zero Issues)

If raw_issue_count == 0: locate plan file (argument path or find docs/ -name "*{SLUG}*"), invoke /stark-plan-to-tasks (3 LLM passes), re-fetch. If raw_issue_count > 0 but all filtered: normal end state, not a decomposition trigger.

Review Round Architecture

Worktree Isolation

Each PR review runs in /tmp/review-{REPO}-pr{N}. Created via git worktree add from FETCH_HEAD. Merge base computed against origin/main. All fixes applied in worktree, pushed to original branch. Cleaned up after loop: git worktree remove + git branch -D.

Round Loop Ownership

This skill manages the round loop, not multi_review.py. multi_review.py runs one round (18 sub-agents: 3 LLMs × 6 domains) and returns JSON. Flags: --json-only (structured output) + --dry-run (don't post to GitHub). The skill classifies findings, fixes, retests, and loops.

Finding Classification

Each finding inspected at file:line in worktree.

fix: severity ≥ medium AND issue exists in code

false_positive: described problem doesn't exist

noise: subjective/style, or 1 agent contradicted by other 2

ignored: below fix threshold (low severity)

Extension Points

Config Hierarchy

.code-review/config.json (repo) → ~/.claude/code-review/config.json (global). Controls: test_command, deploy_command, max review rounds. Repos with custom /stark-release overrides get their workflows respected.

Project Integration

.github/project-config.json enables GitHub Projects V2 integration. Adds: spec completeness gates, status transitions (Ready for Agent → Agent Working → Blocked), AI Suitability filtering. project-pr-sync Action handles post-merge transitions.

Skill Delegation

Delegates to: /stark-plan-to-tasks (decomposition), /stark-release (versioning), /stark-review-improvement (prompt tuning). Each sub-skill respects its own repo-level overrides. The phase executor is an orchestrator, not a monolith.

Observability Protocol

Follows standards/observability.md. TodoWrite for task progress. 5-minute checkpoints. Improvement flags: phase >70% of total time, task fail >30%, agent fail >20%, 0-finding review rounds. Log file consumed by /stark-metrics for cross-phase analytics.

Failure Mode Matrix
Failure Detection Recovery Severity
Dirty working tree git status --porcelain Auto-stash, log warning low
No changes from implementation git diff --stat HEAD empty Log as skipped, continue low
PR creation fails gh pr create exit code Retry once after push; log and continue medium
multi_review.py fails Process exit code / empty JSON Log agent failures, proceed with available findings medium
CI fails before merge gh pr checks --fail-level all Merge with --admin, flag ci_bypassed:true high
Merge conflict git merge/rebase failure Rebase on main, resolve, re-push, retry high
Worktree already exists git worktree add failure Reuse existing: cd /tmp/review-* low
GitHub API rate limit HTTP 429 Wait 60s, retry once; log and continue medium
Subagent timeout Agent tool timeout Log timeout, skip task, continue medium
Release fails /stark-release error Log and skip deploy medium
