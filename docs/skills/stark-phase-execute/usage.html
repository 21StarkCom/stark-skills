> /stark-phase-execute 2026-03-27-observability-v2 --skip-deploy --rounds 2

Common Workflows

Fresh Plan → Full Execution

You have a plan file but haven't decomposed it yet. The skill auto-detects this and runs /stark-plan-to-tasks first, then executes all resulting issues.

/stark-phase-execute docs/plans/2026-03-27-my-feature.md

Resume After Failure

A task failed or your session crashed mid-phase. Use --start-from to resume from a specific issue number. Already-merged tasks are skipped.

/stark-phase-execute my-feature --start-from 47

Dry Run Preview

See which tasks would execute, planned branches, review config, and release type — without creating any branches, PRs, or changes.

/stark-phase-execute my-feature --dry-run

Execute Without Deploy

Implement and release but skip deployment (e.g., staging-only or library projects with no deploy step).

/stark-phase-execute my-feature --skip-deploy
Outputs

Merged PRs

One squash-merged PR per task, with review comments posted by stark-claude[bot]. Each PR closes its corresponding GitHub issue automatically.

Dashboard Report

Printed at the end: task summary table, aggregate stats (duration, findings, fix rate), agent scorecard, and failed task details with recovery suggestions.

GitHub Release

Automatic version bump (patch/minor/major based on task types), CHANGELOG entries, git tag, and GitHub Release with notes. Skippable with --skip-release.

Observability Log

JSON log at ~/.claude/code-review/history/{org}/{repo}/phase-{slug}-{timestamp}.json with per-task timing, findings, and agent performance data.

Key Behaviors
BehaviorDetail
Fully autonomousNo confirmation prompts, no "proceed?" gates. Trigger and walk away.
Sequential tasksEach task merges to main before the next begins — no parallel branches.
Auto-decomposeIf no issues exist for the plan slug, automatically runs /stark-plan-to-tasks first.
Failure isolationFailed tasks are logged and skipped — the phase continues with remaining tasks.
Review in worktreeMulti-agent review runs in an isolated git worktree, cleaned up after each PR.
CI bypassIf CI fails, merges anyway with --admin and flags it in the observability log.
Auth splitUser PAT for PRs/merges, bot tokens only for posting review comments.
