/stark-phase-execute

Autonomous phase execution engine — implement, review, merge, release. Zero intervention.
USAGE GUIDE
Quick Start
$ /stark-phase-execute observability-v2

That's it. The skill fetches all GitHub issues labeled plan:observability-v2, then implements, reviews, and merges each one autonomously.

How to Invoke

By Plan Slug

Use the slug from /stark-plan-to-tasks. Matches the plan:{slug} label on GitHub issues.

/stark-phase-execute 2026-03-api-redesign

By Plan File Path

Pass a path to the plan file. The slug is auto-derived from the filename. If issues don't exist yet, /stark-plan-to-tasks runs automatically.

/stark-phase-execute docs/plans/2026-03-api-redesign.md

Natural Language

The skill triggers on phrases like "execute phase", "run the plan", "autopilot", or "implement this phase".

"execute the observability-v2 phase"

Outputs

Merged PRs

One squash-merged PR per task issue, each closing its GitHub issue automatically via Closes #N.

Review Comments

Multi-agent review findings posted on each PR via stark-claude[bot]. Findings classified as fix, noise, or false positive.

Release

CHANGELOG entries, semantic version bump, git tag, and GitHub Release with notes. Delegates to /stark-release.

Observability Log

JSON file in ~/.claude/code-review/history/{org}/{repo}/ with per-task timing, findings, and agent stats.

Dashboard

Terminal summary with task table, aggregate stats, agent scorecard, and improvement flags.

Improvement Flags

Signals for prompt tuning: high FP rates, repeated finding types, agent blind spots, unparseable output.

Common Usage Patterns

Preview Before Running

See what would happen without making changes. Shows task list, branches, review config, and release plan.

/stark-phase-execute my-plan --dry-run

Resume After Failure

Skip already-merged tasks and resume from a specific issue number.

/stark-phase-execute my-plan --start-from 47

Plan File → Full Execution

Pass a plan file directly. If issues don't exist yet, /stark-plan-to-tasks runs first, then execution begins.

/stark-phase-execute docs/plans/2026-03-auth-rewrite.md

Skip Release Step

Execute tasks and reviews but don't bump version or deploy. Useful for WIP phases.

/stark-phase-execute my-plan --skip-release

More Review Rounds

Increase review thoroughness for high-risk phases (default is 3).

/stark-phase-execute my-plan --rounds 5

Cross-Repo Execution

Target a different repo than the current working directory.

/stark-phase-execute my-plan --repo GetEvinced/infra-pulse

Prerequisites

CLI Tools in PATH

gh, claude, codex, gemini must all be installed and accessible.

GitHub Auth

gh auth status must show an active PAT. GitHub Apps (stark-claude, stark-codex, stark-gemini) installed on the target repo.

Clean Working Tree

Must be on main with no uncommitted changes. The skill validates this at startup.

Failure Handling
FailureBehavior
Task implementation produces no changesLogged as skipped, moves to next task
PR creation failsRetries once after re-push; if still fails, logs and continues
CI checks fail before mergeMerges anyway with --admin, flags ci_bypassed in log
Multi-agent review dispatch failsProceeds with available findings from working agents
Merge conflictRebases on main, resolves, re-pushes, retries merge
Test suite fails (regression)Logs failures, continues to release/dashboard
Release failsLogs error, skips deploy
Any unrecoverable errorCleans up branch/worktree, logs full error, continues to next task
