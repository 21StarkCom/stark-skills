Deep Dive: The Autonomous Task Loop

1. Implementation Subagent

A fresh branch is cut off main for each issue. An internal subagent explores the workspace, reads the issue requirements, writes the code, runs local tests, and creates semantic commits (e.g., feat(scope): description (#NUM)).

2. Multi-Agent Review Loop

Creates PR and immediately spawns a multi-agent review team via an isolated git worktree. Findings are classified by severity. The skill autonomously fixes fix-level findings, pushes the updates, and repeats up to --rounds times.

3. Autonomous Merge

Once the multi-agent review passes (0 findings or only ignored noise), it waits for GitHub CI checks. If CI passes, it instantly squash-merges into main and cleans up branches. No human approval gates.

Outputs & Dashboards

Terminal Dashboard

Once the entire phase completes, the terminal will render a full scorecard including:

  • Task Summary Table: Time taken, PR link, review finding counts per issue.
  • Agent Scorecard: How Claude, Codex, and Gemini performed (accuracy, noise rate, unique findings).
  • Release Details: CHANGELOG entries added, bump level, deploy status.

Housekeeping & Feedback

The system constantly learns from execution:

  • Project Memory: Automatically records decisions and major structural changes.
  • Prompt Improvements: Analyzes review loop metrics. If an agent had >20% false positive rate or missed obvious bugs, it suggests running /stark-review-improvement.
