Failure Modes & Recoveries

Failure LLM calls fail

Impact: Specific visualizations fail to generate.

Recovery: Report which LLM failed and continue processing the remaining skills or alternative models.

Failure Playwright missing

Impact: Cannot generate screenshot visualizations.

Recovery: Skip screenshot generation entirely and output a warning to the user.

Failure No changes detected

Impact: User requests generation but all docs are synchronized.

Recovery: Safely exit script with "all up to date" report. User can override with --force.

CLI Arguments
Argument / Flag Behavior & Purpose
(No args) Regenerate docs for skills that have changed SKILL.md files.
--skill <name> Regenerate documentation for one specific skill explicitly. Skips detection.
--all Regenerate all skill documentations (alias for --force).
--check Check if any documentation is stale relative to its SKILL.md. Returns exit status without generating.
