Quick Start
# Regenerate docs for skills with changed SKILL.md files
> /stark-generate-docs

# Regenerate docs for one specific skill
> /stark-generate-docs --skill stark-review

# Regenerate all skill docs
> /stark-generate-docs --all

# Check if any docs are stale (dry run)
> /stark-generate-docs --check
Arguments
Flag Description Example
(none) Auto-detect changed SKILL.md files and regenerate only those /stark-generate-docs
--skill <name> Regenerate docs for a single named skill /stark-generate-docs --skill stark-review
--all Force regenerate all skill docs (alias for --force) /stark-generate-docs --all
--force Force regenerate all skill docs /stark-generate-docs --force
--check Check for stale docs without making changes /stark-generate-docs --check

Generated Artifacts

usage.html / internals.html

Standalone HTML pages with inlined CSS. Two perspectives per skill: usage (end-user) and internals (developer). Multi-LLM tournament selects the best visualization.

usage.md / internals.md

Markdown documentation with embedded Mermaid diagrams, prerequisites, quick start examples, common patterns, and troubleshooting sections.

usage.mermaid / internals.mermaid

Standalone Mermaid flowchart files showing the skill's workflow. Renderable in GitHub, MkDocs, or any Mermaid-compatible viewer.

usage.json / internals.json

Structured JSON with prerequisites, quick start, common patterns, troubleshooting, related skills, and argument tables for programmatic consumption.

usage.png / internals.png

PNG screenshots of the HTML visualizations via Playwright. Used as preview images in documentation indexes and READMEs.

scores.jsonl

Audit trail of tournament evaluations. Records each LLM's score per skill per perspective, enabling quality tracking over time.

Common Workflows

After Editing a Skill

Modify a SKILL.md, then run /stark-generate-docs with no arguments. It auto-detects the changed skill and regenerates only its docs.

Full Regeneration

Run /stark-generate-docs --all to regenerate every skill's docs. Useful after changing the CSS design system or doc templates.

CI Staleness Check

Run /stark-generate-docs --check in CI to verify no docs are stale. Returns exit code 1 if any need regeneration.
