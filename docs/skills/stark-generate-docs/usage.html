stark-generate-docs

\n
Generate or refresh skill documentation from changed SKILL.md files, then commit the updated docs.
\n
USAGE VIEW
\n

\n This skill is for end users who want a fast way to regenerate skill docs, target one skill, check for stale docs without changing files, or force a full refresh.\n

Quick Start
\n
\n
\n

Default Invocation

\n

Run /stark-generate-docs to regenerate docs only for skills whose SKILL.md files are stale or changed.

\n
\n
\n

Target One Skill

\n

Run /stark-generate-docs --skill <name> when you know exactly which skill needs refreshed docs.

\n
\n
\n

Safety Check

\n

Run /stark-generate-docs --check to detect stale docs without generating files or committing anything.

\n
\n
\n

Force Full Refresh

\n

Run /stark-generate-docs --all or --force to rebuild documentation for every skill.

\n

Common Commands
\n \n \n \n \n \n \n \n \n \n \n \n \n \n \n \n \n \n \n \n \n \n \n \n \n \n \n \n \n \n \n
CommandWhen to Use ItExpected Result
/stark-generate-docsNormal workflow after editing one or more SKILL.md files.Only stale skill docs are regenerated and committed.
/stark-generate-docs --skill stark-generate-docsYou want to refresh docs for exactly one named skill.That skill's docs are rebuilt without scanning the rest.
/stark-generate-docs --checkYou need a non-destructive stale-doc check in CI or before generating.Reports up-to-date vs stale; no docs or commits are created.
/stark-generate-docs --allYou want a full refresh across the skill catalog.All skill docs are regenerated, equivalent to --force.

Inputs And Outputs
\n
\n
\n

Inputs

\n

Command flags, skill names, repo state, and which SKILL.md files changed.

\n
\n
flag --skill <name>
\n
flag --all / --force
\n
flag --check
\n
\n
\n
\n

Behavior

\n

Detect stale docs when needed, regenerate selected skills, and continue even if some LLM steps fail.

\n
\n
phase Detect changes
\n
phase Generate docs
\n
phase Commit results
\n
\n
\n
\n

Outputs

\n

Updated documentation files, a git commit, and a final status report with winners, scores, and file counts.

\n
\n
Winners + scores
\n
artifact docs/skills/
\n
report updated skill list
\n
\n
\n
\n\n
Failure Modes
\n
\n
\n

LLM Calls Fail

\n

Generation should report which model calls failed and continue processing remaining work instead of aborting the entire run.

\n
\n
\n

Playwright Missing

\n

Screenshot generation is skipped. The user still gets docs, but with a warning about missing screenshots.

\n
\n
\n

No Changes Found

\n

If the check exits cleanly, report that all skill docs are up to date and stop without generating new files.
