stark-generate-docs

Generate or refresh skill documentation from changed SKILL.md files, then commit the updated docs.
USAGE VIEW

This skill is for end users who want a fast way to regenerate skill docs, target one skill, check for stale docs without changing files, or force a full refresh.

Quick Start

Default Invocation

Run /stark-generate-docs to regenerate docs only for skills whose SKILL.md files are stale or changed.

Target One Skill

Run /stark-generate-docs --skill <name> when you know exactly which skill needs refreshed docs.

Safety Check

Run /stark-generate-docs --check to detect stale docs without generating files or committing anything.

Force Full Refresh

Run /stark-generate-docs --all or --force to rebuild documentation for every skill.

Common Commands
Command When to Use It Expected Result
/stark-generate-docs Normal workflow after editing one or more SKILL.md files. Only stale skill docs are regenerated and committed.
/stark-generate-docs --skill stark-generate-docs You want to refresh docs for exactly one named skill. That skill's docs are rebuilt without scanning the rest.
/stark-generate-docs --check You need a non-destructive stale-doc check in CI or before generating. Reports up-to-date vs stale; no docs or commits are created.
/stark-generate-docs --all You want a full refresh across the skill catalog. All skill docs are regenerated, equivalent to --force.

Failure Modes

LLM Calls Fail

Generation should report which model calls failed and continue processing remaining work instead of aborting the entire run.

Playwright Missing

Screenshot generation is skipped. The user still gets docs, but with a warning about missing screenshots.

No Changes Found

If the check exits cleanly, report that all skill docs are up to date and stop without generating new files.
