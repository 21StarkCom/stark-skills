Architecture Overview

Entry Point

Skill invoked via /stark-pr-status with optional PR number, flags for repo, state, output format. Arguments parsed and forwarded to pr_status.py.

Data Engine

pr_status.py in ~/.claude/code-review/scripts/ — Python script run via the scripts venv. Merges two data sources: GitHub API (live) and stark-review history (local JSONL).

Output Modes

Two rendering paths: single-PR (detailed dashboard) and all-PRs (summary table). Both support --json for machine-readable output.

Observability Metrics

PRs Analyzed

Count of PRs processed in this invocation. Single-PR mode = 1, all-PRs mode = up to --limit.

GitHub API Calls

Number of REST API requests made. Varies by mode — single PR needs ~4 calls (PR, reviews, comments, timeline), all-PRs uses list endpoint + per-PR detail.

History Records Loaded

Count of stark-review history JSONL records matched to the queried PR(s). Zero if no review history exists.

Data Sources

Enum: github-only (no history), history-only (offline), or both (full fusion). Determines which dashboard sections populate.

Failure Modes & Recovery
Exit Code Failure Recovery
1 No repo detected Run from a git repo or pass --repo org/name
2 PR not found Verify PR number exists in the target repo
3 GitHub API error Run install.sh or check GitHub App credentials in Keychain
— No history for PR Graceful degradation: show GitHub data only, note "No stark-review history"
3 Rate limited Backoff message: "GitHub API rate limited. Try again in a few minutes."
Extension Points

pr_status.py CLI Flags

New flags added to the Python script are automatically available — the skill forwards all arguments via $ARGUMENTS. No skill-side changes needed for new filters.

Suggestion Heuristics

Phase 3 pattern matching lives in the skill prompt, not the script. Add new heuristics by appending rules to the "Actionable Suggestions" section — e.g., "PRs with >5 review rounds → suggest splitting".

Data Source Fusion

The script merges GitHub API and local history. To add a third source (e.g., CI metrics), extend pr_status.py data loading and add a new observability enum value.

Output Formats

Currently supports terminal text and --json. New formats (CSV, HTML) would be added in the script's rendering layer. The skill's Phase 2 prints output verbatim regardless of format.
