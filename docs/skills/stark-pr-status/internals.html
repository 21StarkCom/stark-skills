Primary Job

Explain what happened on a PR, not just its current state. The skill reconstructs review rounds, severity distribution, outcome quality, participant behavior, and merge timing.

Data Blend

GitHub provides canonical PR, review, and participant events. stark-review history adds findings, noise classification, and historical review signal that GitHub alone does not expose.

github-only history-only both

Presenter Contract

The Python script owns formatting. Single-PR mode emits a rich dashboard; all-PR mode emits a compact table; --json exposes a machine-readable variant for downstream consumers.

terminal-first skill

Runtime Contract

Invocation Surface

Entry point is pr_status.py with positional PR selection or --all. Repo detection defaults to the current git checkout unless --repo org/name is supplied.

Mode Switches

--state open|closed|merged|all constrains the PR set, --limit N caps fleet summaries, and --json bypasses human-oriented rendering for structured output.

Post-Analysis Heuristics

After rendering, the skill inspects stale PRs, high-noise patterns, and merge-ready conditions to suggest adjacent skills such as /stark-review, /stark-review-improvement, and /stark-pr-flow.

Failure Modes And Recovery

No Repo Detected

Exit 1. Triggered when the current directory is not a git repo and no --repo override exists. Recovery is explicit: run from a repo or pass --repo org/name.

PR Not Found

Exit 2. Single-PR lookups fail fast when the requested PR number does not exist in the selected repo. The UI should preserve the repo context in the error text.

GitHub API Failure

Exit 3. Covers auth failures, upstream API issues, and rate limiting. GitHub state is canonical, so the skill cannot safely reconstruct the dashboard without it.

No History For PR

Non-fatal. The skill still shows GitHub-derived review activity and clearly notes that stark-review history is unavailable, preventing silent gaps in analytics quality.

Observability And Extension Points

Observability Protocol

Emit the shared Skill Observability Protocol and add skill-specific counters for PRs analyzed, GitHub API calls, history records loaded, and whether the result was sourced from GitHub, history, or both.

Analytics Extension Point

New metrics should land in the lifecycle aggregator first. That keeps single-PR dashboards, fleet summaries, and JSON output aligned instead of duplicating business logic across presenters.

Presenter Extension Point

If you need richer automation, extend --json rather than scraping terminal output. The terminal contract remains stable because the script already owns preformatted display text.

Workflow Extension Point

Heuristic suggestions are intentionally isolated. Add org-specific rules for stale reviews, reviewer load, or CI gating without touching collection or normalization logic.
