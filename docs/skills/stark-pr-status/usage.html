Input Arguments & Flags

Target Arguments

  • PR_NUMBER: Specific PR (e.g., 15)
  • --all: All PRs (default if no number)
  • --repo org/name: Override repo detection

Filtering & Format

  • --state: open, closed, merged, all (default: all)
  • --limit N: Max PRs to show (default: 20)
  • --json: Machine-readable JSON output
Dashboard Output Features (Single PR)

Header & Timeline

Title, status, author, dates, time-to-merge, and key events in chronological order.

Review Rounds

Count of review cycles, findings per round, and the improvement delta over time.

Findings Analysis

Breakdown by severity. Analysis of outcome: fix vs. noise vs. false-positive vs. ignored.

Participants & Signal

Humans + bots with action counts. Identifies most impactful finding and biggest noise source.

Actionable Suggestions Logic
Condition Detected Suggested Action / Routing
PR open > 7 days with no reviews Suggest invoking /stark-review
PR has a high noise ratio (false positives/ignored) Suggest invoking /stark-review-improvement
PR is ready to merge (Approved, CI green) Suggest invoking /stark-pr-flow to finalize
Failure Modes & Recovery

Missing Context

No repo detected: Prompt user to run from a git repo or provide --repo org/name.

PR not found: Notify user that PR #N was not found in the current repository.

API & Data Issues

Auth/API Failure: Suggest running install.sh or checking GitHub App credentials. If rate limited, wait a few minutes.

No local history: Show GitHub data only and note "No stark-review history".
