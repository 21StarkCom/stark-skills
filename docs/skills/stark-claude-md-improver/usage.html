Analysis Dimensions

1. Structure & Clarity

Logical organization, scannable headers, no redundancy across hierarchy levels. Each level should add specificity, not repeat parent info.

2. Completeness

  • Project overview & tech stack
  • Architecture & key files
  • Dev workflow & env vars
  • Conventions, gotchas, don'ts

3. Accuracy & Freshness

Cross-references actual project state. Flags missing files, outdated versions, broken commands, and stale URLs.

4. Effectiveness for AI

Actionable instructions ("Use X" not "We typically use X"), specific conventions, no conflicting or implicit assumptions.

5. Hierarchy Optimization

Content at the right level — personal prefs in ~/CLAUDE.md, org in org/CLAUDE.md, project specifics in project/CLAUDE.md.

Key Rules

Don't Over-Add

Only add sections that genuinely help. No content for completeness sake. CLAUDE.md is for AI instructions, not human documentation.

Match Existing Style

Suggested content uses the same tone and formatting as the existing file. Respects user's established conventions.

Don't Duplicate Specs

If a CLAUDE.md references a spec or doc, the skill verifies the reference — it doesn't inline the content.
