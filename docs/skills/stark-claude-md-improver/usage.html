The 5 Analysis Dimensions

1. Structure & Clarity

Is the information logical and scannable? Checks for appropriate headers and ensures there is no redundancy across the file hierarchy.

2. Completeness

Checks for critical missing elements that help Claude function effectively.

  • Overview & Tech Stack
  • Architecture & Key Files
  • Dev Workflow & Env Vars
  • Conventions, Gotchas & Don'ts

3. Accuracy & Freshness

Cross-references instructions with actual project state. Flags non-existent files, outdated versions, broken commands, and dead URLs.

4. Effectiveness for AI

Are the instructions actionable? Highlights vague conventions (e.g., "follow best practices") and identifies conflicting instructions across the hierarchy.

5. Hierarchy Optimization

Evaluates whether information belongs at the ~/ (personal), org/, or project/ level. Suggests content promotions and demotions to reduce duplication.

Core Operating Rules

Content Value

Do not add sections simply for completeness. Only add content that genuinely helps Claude Code. Prefer concise instructions over verbose documentation.

Tone & Style

Respect the user's existing style and tone. When suggesting new content, seamlessly match the surrounding text.

AI vs Human

Do not add README-style content. CLAUDE.md is strictly for AI consumption and behavior steering, not human onboarding.
