Analysis Dimensions (scored 1–5 per file)

1. Structure & Clarity

Logical organization, scannable headers, no redundancy across hierarchy levels. Each level should add specificity, not repeat parent info.

2. Completeness

Checks for: project overview, tech stack, architecture, key files, dev workflow, env vars, conventions, gotchas, don'ts. Only flags genuinely missing sections.

3. Accuracy & Freshness

Cross-references with actual project state. Flags: missing files, stale versions, broken URLs, non-functional build commands, dead key-file paths.

4. AI Effectiveness

Actionability: "Use X" beats "We typically use X". Specificity: concrete naming rules over "best practices". Checks for conflicting or implicit instructions.

5. Hierarchy Optimization

Right content at right level: personal prefs in ~/CLAUDE.md, org config in org/CLAUDE.md, project specifics in project/CLAUDE.md. Suggests promotions/demotions.

Output Report Structure

Issues Found

Concrete problems with file:line references. Stale paths, broken commands, conflicting instructions, redundant content across levels.

Suggested Additions

Missing content that would genuinely help AI effectiveness. Includes full draft text written in the existing file's style and tone.

Suggested Removals

Redundant or outdated content. Items duplicated from parent CLAUDE.md levels, references to removed features, stale version numbers.

Suggested Moves

Content at the wrong hierarchy level. Personal prefs in project file → promote to ~/CLAUDE.md. Org-wide patterns in project → promote to org.
