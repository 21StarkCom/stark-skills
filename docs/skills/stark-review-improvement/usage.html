Input Sources

Conversation Context (Preferred)

Run /stark-review-improvement in the same session as /stark-review. The assessment is already in context — no file lookup needed.

History Files (Fallback)

If invoked in a new session, the skill scans ~/.claude/code-review/history/ for the most recent prompt-assessment.md file.

Common Workflow

1. Run a Review

Start with /stark-review 42 to run a multi-agent PR review. The assessment at the end identifies prompt quality issues.

2. Improve Prompts

In the same session, invoke /stark-review-improvement. Review the proposed action items, select which to apply.

3. Verify & Ship

Review the git diff, confirm the commit. Use /stark-pr-flow to open a PR with the prompt improvements.

Outputs

Modified Prompts

Targeted edits to agent.md and domain prompt files across Claude, Codex, and Gemini directories.

Prompt Changelog

docs/prompt-changelog.md — tracks every improvement with source PR, rationale, and change table.

Git Commit

Structured commit message with source attribution and change counts by category.

Safety Constraints

Never Rewrites Entire Prompts

Only minimal, targeted edits. Existing structure and instructions are preserved. Instructions are added, never removed.

Backward Compatible

New config fields always have empty/null defaults. Prompt changes don't break existing output formats.

User Reviews Before Commit

You see the full git diff and confirm before any changes are committed. No auto-push.
