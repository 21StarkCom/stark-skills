stark-review-improvement

Closes the feedback loop by translating prompt improvement assessments into targeted code updates.

Complexity: Medium

How to Invoke

Trigger this skill by providing a natural language instruction or the specific slash command:

  • /stark-review-improvement
  • "improve review prompts"
  • "fix review prompts based on the assessment"

Note: The skill automatically reads the assessment from the current conversation context or the latest history in ~/.claude/code-review/history.

Change Targets & Mechanics

Prompt Edits

Modifies $PROMPTS/{agent}/*.md. Focuses on minimal, section-specific fixes without rewriting the entire prompt.

Scope Fixes Output Formats False Positives

Orchestrator Edits

Modifies $SCRIPTS/multi_review.py. Adjusts how variables are passed or how output is post-processed.

Variable Injection File Deduplication

Config Edits

Modifies config.json (Global/Org). Safely adds new configuration arrays or objects with backward-compatible defaults.

Disabled Paths Severity Overrides

Important Constraints

Constraint Description
No Full Rewrites Never rewrite an entire prompt file. All prompt edits must be surgical and targeted.
Additive Constraints Never remove existing instructions. Add constraints to fix issues, do not delete core capabilities.
Backward Compatibility New config fields must have empty/null defaults. multi_review.py changes must default safely.
Separation of Concerns Diff scoping instructions strictly go in agent.md, while domain-specific criteria stay in domain prompts.
Explicit Confirmation User must always review the final git diff before the commit is generated. The agent will never git push autonomously.
