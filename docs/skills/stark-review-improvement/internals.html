Edit Implementations

Prompt Edits

Target: $PROMPTS/{agent}/*.md

Rules: Minimal changes. Retain headings and structure. If adding exclusions, add to the existing matching section rather than creating new sections. agent.md handles diff scoping; domain prompts handle domain logic.

Orchestrator Edits

Target: $SCRIPTS/multi_review.py

Rules: Surgical edits only (e.g., injecting variables like {base} or fixing deduplication). Requires adding tests if logic changes. Must maintain backward compatibility.

Config Edits

Target: config.json (Global/Org)

Rules: Add new fields (e.g., disabled_paths) with empty/null defaults. Document inline or in README. Ensure global schema supports the organization overrides.

Engine Constraints & Observability

Core Constraints

1. Never rewrite entire files; targeted surgical edits only.

2. Never remove instructions; add constraints instead.

3. One concern per edit. Do not bundle unrelated improvements.

4. Diff Review Required: Show git diffs prior to finalizing local commit.

Skill Metrics (Observability)

Standard timing and checkpoint tracking from observability.md, plus:

  • Validation: syntax checks passed vs failed.
  • Files modified: count by type (prompt, orchestrator, config).
