Architecture View

Data Sources and Resolution Order

The skill prefers in-memory context because it preserves the current run's assessment without requiring filesystem lookup. If absent, it falls back to the newest history snapshot.

1. Conversation

Find a “Prompt Improvement Assessment” section or a referenced prompt-assessment.md path.

2. History Fallback

Resolve the latest history directory and read prompt-assessment.md there.

3. Hard Stop

If neither source exists, the skill exits with an explicit prerequisite error.

Mutation Boundaries

Each edit type maps to a narrow ownership area so the system can evolve without conflating prompt behavior, orchestration behavior, and operator policy.

Prompt Layer

Agent wording, diff scoping, output-format rigidity, exclusions, and severity guidance.

Orchestration Layer

Context injection, filtered diff delivery, and post-parse dedup logic.

Config Layer

Persistent knobs such as disabled paths and severity overrides, kept backward compatible.

Extension Points

Assessment Parsing

To support richer assessment formats, extend the row-to-action-item parser. Preserve the current output contract: pattern, recommendation, target, and category.

Phase 1.2 New classifiers should map cleanly to prompt/orchestrator/config/no-action.

Prompt Policy Tuning

Most behavior changes should land in agent prompt files, especially diff scoping, exclusions, and JSON-only output hardening.

agent.md scope and format
domain prompt domain-specific false-positive suppression

Review Orchestration

When a defect is structural rather than instructional, patch the orchestrator instead of overloading prompts. Keep changes surgical and test logic changes.

_run_subagent() base ref injection and filtering
_parse_findings() downstream dedup hook

Operational Memory

The changelog and copied assessment create institutional memory. This is the durable bridge from one review run to later prompt quality improvements.

docs/prompt-changelog.md human-readable ledger
history/.../prompt-assessment.md reproducible source artifact
Failure Modes and Safeguards

Missing Assessment

Stops before any edit. Recovery is external: run /stark-review first or provide the assessment path in context.

Misplaced Fix Type

The classification step prevents scope leakage, keeping scoping rules in agent.md and domain rules in domain prompts.

Backward Compatibility

New config keys require empty defaults. Orchestrator changes should preserve existing behavior when settings are absent.

Internal Operating Rules

Non-Negotiable Constraints

Never rewrite a full prompt, never remove existing instructions, keep one concern per edit, and always show diffs before committing. The intent is traceable, low-risk evolution rather than prompt churn.

Outputs Produced

Approved edits, validation results, prompt changelog entry, optional history copy, and a local commit summarizing where the improvement signal came from.
