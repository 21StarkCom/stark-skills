Architectural Principles

Dual-Auth Boundaries

The workflow intentionally splits authentication modes to prevent bot permission escalation:

  • User PAT: Creating PRs, merging PRs (bypasses branch protections).
  • Bot Token: Scoped exclusively to reading code diffs, querying project state, and posting PR comments natively as stark-claude[bot].

Extension & Hook Points

Key areas intended for internal developers to modify or extend:

  • stark_claude.py: The diff analysis and inline-commenting engine.
  • github_projects.py: Advisory check validations against custom org-level project properties (e.g., Documentation State).

Observability Contract

Complies with observability.md, generating telemetry metrics for:

  • Push duration and conflict hit rate.
  • PR creation and auto-generation token usage.
  • Self-review execution duration.
  • Admin bypass telemetry via --admin merge strategy.

Failure Modes & Recovery Matrix

Failure Condition Detection Mechanism Automated Recovery / Manual Action Required
Dirty Tree / Main Branch Pre-flight checks using git status and git branch Abort execution. Requires user to commit, stash, or branch out.
Git Push Diverged Exit code from git push -u origin Automated recovery: Execute git fetch && git rebase origin/main. Requires manual conflict resolution if rebase fails.
GH Auth Failure Exit code from gh pr create Verify gh auth status. Ensure user's PAT is active, abort if token is invalid.
Bot Script Failure Exception in Python virtual environment Check dependencies (PyJWT, requests, cryptography). Ensure execution path uses scripts/.venv/bin/python3.
