stark-pr-flow

End-to-end PR workflow for GetEvinced repos: push -> PR -> self-review -> merge.
Target Audience: End User (Usage)

How to Invoke

Trigger Phrases

"open PR", "create PR", "merge this", "ship it", or simply "stark-pr-flow".

Arguments (Optional)

<title override>: Specify a custom PR title.
draft: Include the exact word to create the PR as a draft.

Prerequisites

You must be on a feature branch (not main) with a clean working tree (no uncommitted changes).

Execution Flow

Start Command
Triggered with optional args (title, draft)
Check Pre-reqs Aborts if on `main` or if working tree is dirty.
Step 1: Prepare Branch
git push -u origin $BRANCH
Analyze Changes Generates conventional PR title & summary based on commits and diff.
Step 2 & 3: Create PR
gh pr create (with PAT)
Artifact Creates Pull Request (Draft only if requested).
Bot Auth Uses BOT_TOKEN to post review as stark-claude[bot].
Step 4: Self Code Review
Analyzes diff and posts structured findings via stark_claude.py
Review Comment
Issues, Notes, & Inline Comments
Advisory Check Warns if issue doc state is "Not Started" or "Drafted".
Step 5: Present Summary — STOP
Wait for explicit user approval to merge
User Input Required e.g., "yes", "merge it", "reject"
Step 6: Merge & Clean Up
gh pr merge --squash --admin
Cleanup Checks out main, pulls, deletes local/remote feature branch.

Common Failure Modes & Recovery

Failure State System Behavior / Recovery
Pre-req Fail On main / Dirty tree Aborts immediately. Prompts user to create a branch, or commit/stash changes first.
Git Push diverged Aborts. User must git pull --rebase origin main, resolve conflicts, then push.
Auth gh auth fails for PR Verify gh auth status. The user's Personal Access Token must be active for PR operations.
CI/CD PR checks failing Shows check status to user. Asks whether to force-merge with --admin or wait/fix first.
Rejection User rejects PR Leaves PR open for revisions, or closes it based on user instruction. Does not merge.
