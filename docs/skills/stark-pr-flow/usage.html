Quick Start Examples

Basic — Ship Current Branch

Just type /stark-pr-flow on any feature branch. The skill auto-detects the repo, generates a title from your commits, creates the PR, posts a self-review, and waits for your go-ahead to merge.

Custom Title

/stark-pr-flow feat: add retry logic to webhook handler
Overrides the auto-generated PR title with your own. Useful when commit messages don't capture the full picture.

Draft PR

/stark-pr-flow draft
Creates the PR as a draft for early feedback. The self-review still posts, but you control when to mark it ready.

What Happens at Each Step

Auth Handling

Automatically manages the auth split: your personal GitHub token (via gh) creates and merges the PR so it shows as you. The bot token posts the review so it appears as stark-claude[bot]. You never touch tokens.

Self-Review Quality

Every changed line is reviewed for: missing error handling, type hints, import organization, test gaps, hardcoded values, and security concerns. Findings are posted as structured comments on the PR.

Merge Strategy

Always squash-merge with --admin to bypass branch protection. Your feature branch is deleted both locally and on the remote. Main is pulled and rebased automatically.

Key Details

PRs Are NOT Draft

By default, PRs are created as ready for review. Only pass "draft" explicitly to create a draft PR.

Always Waits for Approval

The skill never merges automatically. You must explicitly say "yes", "merge it", "go ahead", or "ship it".

Doc State Advisory

Before merge, checks the linked issue's Documentation State in GitHub Projects. Warns if incomplete but never blocks.
