Architecture Components

Python Processing Layer (metrics.py)

A decoupled backend responsible for data ingestion. It robustly handles legacy or corrupted JSON files via schema normalization and gracefully skipping unparseable records. It outputs terminal-ready text or structured JSON based on CLI flags.

Interactive Shell Wrapper

The bash layer acting as the entry point. It not only pipes arguments to the python script but also analyzes the string output to drive an interactive terminal loop (Phase 2), prompting users to take direct action on generated recommendations.

Meta-Observation Engine

A lightweight analysis step in the wrapper (Phase 3) that cross-references current data with previous states to flag critical insights. It detects system-wide improvements ("wins" like lower FP rates) or regressions (climbing agent failure rates).

Custom Telemetry Schema

Adheres to the Skill Observability Protocol. Custom fields include records_loaded (by type), filters_applied (repo, skill, date), sections_generated, and recommendations_count to track usage patterns internally.
