Architecture Notes

Execution boundary

The markdown skill does not compute metrics itself. Its responsibility is stable command construction, exit-code interpretation, and precise operator messaging around the Python report generator.

Best extension seam

New analysis dimensions belong in metrics.py. The skill wrapper should only change when adding a new flag, a new post-report action, or a new operator prompt tied to recommendations.

Decision ownership

There are two real branches: process exit handling and recommendation follow-up. Everything else is linear flow, which keeps the terminal interaction predictable and easy to debug.

Output contract

The skill explicitly trusts script formatting. That avoids dual rendering logic, but it also means presentation changes should be made in the script so JSON and terminal text stay aligned.

Data assumptions

History input may span multiple formats and partial corruption. The system tolerates bad files by warning on stderr, skipping them, and continuing aggregation over the remaining records.

Operational risk

If agent failure rates climb, the skill must elevate urgency after report presentation. That makes this skill part reporting layer and part early-warning surface for review-system health.

Observability + Modification Guide

Must-measure

Records loaded by skill type, filters applied, sections generated, and recommendation count.

Trend signals

Lower false-positive rate after prompt changes counts as a visible win; rising failure rates should trigger urgent wording.

Change safely

Add new report sections in the script first, then expose any new interaction branch in the wrapper.

Contributor edit checklist

1. Add or adjust parsing logic in metrics.py if history format changes.
2. Update flag passthrough only if a new operator control is needed.
3. Keep exit-code semantics stable so downstream usage docs remain valid.
4. Extend post-report action routing when new remediation skills exist.

Interpretation

Why this skill is simple

At roughly 74 lines, it behaves as a disciplined adapter: minimal branching, explicit messaging, and strong reliance on a single downstream script.

Why it is still important

It turns historical review data into an operator loop. Without this skill, contributors lack a standard path to inspect review quality, runtime regressions, and prompt-change impact.

What to watch next

Recommendation loop maturity is the next leverage point: the more precisely recommendations map to concrete remediation actions, the more this becomes an active tuning console rather than a passive report viewer.
