Arguments & Filters
Argument Usage Example Description
--repo REPO --repo Evinced/stark-skills Filters metrics to a specific repository. Useful for analyzing performance drops on particular codebases.
--skill SKILL --skill stark-review Isolates metrics for a single skill to track its accuracy and execution time independently.
--since DATE --since 2026-03-01 Limits historical data to runs on or after the specified date. Essential for tracking "before and after" prompt improvements.
--json --json Disables interactive prompts and terminal formatting, outputting raw JSON. Ideal for programmatic ingestion or CI scripts.
Key Capabilities

Terminal Analytics

Translates complex code-review history into immediately readable Agent Scorecards, tracking false positive rates, duration trends, and overall finding quality across runs.

Actionable Tuning

Doesn't just display data; it actively reads the "Recommendations" section and prompts the user to apply fixes (e.g., dispatching /stark-review-improvement automatically).

Meta-Observations

The skill intelligently detects system-wide health trends. It will congratulate you if recent prompt changes lowered FP rates, or flag urgent warnings if agent failure rates are spiking.

Common Failure Modes
Symptom / Failure Agent Output & Recovery Path
No history directory Recovery: "No history found. Run /stark-review to generate data."
No matching records Recovery: "No records match filters. Try broader criteria." (Usually caused by overly restrictive --since or --repo).
Corrupt JSON in history Recovery: Script warns on stderr, safely skips the corrupted file, and continues processing the rest.
Script not found Recovery: "Run install.sh to set up stark-skills"
