Design Principles & Rules

Version Source of Truth

Git Tags dictate the official version. pyproject.toml remains static at 0.1.0 to avoid trivial conflicts. src/infra_pulse/__init__.py serves as the runtime mirror that MUST be bumped so the deployed app correctly reflects the tag.

Changelog Integrity

The system requires developers to maintain standard ## [Unreleased] sections in CHANGELOG.md using ### Fixed, ### Added, and ### Changed markers. This powers automated semantic bumping without requiring human prompts.

Fail-Closed Executions

The skill refuses to run if uncommitted changes exist, if not on main, if the CHANGELOG is missing, or if the Unreleased block is empty. This prevents accidental partial releases or empty tags.

Failure Modes & Recovery

Failure State Detection Phase Recovery / Action taken
Not on main branch 1. Pre-flight Abort. Prompt user to git checkout main.
Dirty working tree 1. Pre-flight Abort. Prompt user to stash or commit changes first.
No CHANGELOG.md / Empty 3. Review Changes Abort. Nothing to release. Instruct user to run /fix-bug or add logs.
Tag already exists 7. Create Tag Error out. Suggest next logical version.
Push fails 8. Push Attempt git pull --rebase origin main and retry push.
gh auth fails 9. GH Release Verify via gh auth status. Fails if user PAT is inactive.

Observability Events

Skill Observability Protocol (~/.claude/code-review/standards/observability.md)

The skill emits standard checkpoints and timing data plus domain-specific tags:

  • Version Metadata: Previous version, new version, bump type (patch | minor | major).
  • Changelog Analytics: Count of entries categorized by Fixed, Added, Changed.
  • Artifact Creation: Booleans for tag created (yes/no) and GH release created (yes/no).
  • Latency: Measure total git push duration to monitor remote delays.
