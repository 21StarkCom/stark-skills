Important Rules & Conventions

Version Source of Truth

Git tags are the primary source of truth. The fallback baseline is 0.1.0 if no tags exist. The static version in pyproject.toml is ignored and never bumped.

Runtime Version Check

The system must bump src/infra_pulse/__init__.py alongside the Git tag. If skipped, the deployed application will display an outdated version to end users.

GitHub Authentication

Releases must appear as authored by the user, not a bot. Ensure unset GH_TOKEN is run so the native gh CLI uses the user's active Personal Access Token (PAT).

Changelog Hygiene

Never release an empty [Unreleased] section. All items must be moved into the new versioned section, leaving an empty ## [Unreleased] header for future PRs.

Common Failure Modes & Recovery

Failure Mode Description Recovery Strategy
Not on main branch Skill triggered on a feature branch. Prompt user to run git checkout main && git pull.
Dirty Working Tree Uncommitted changes present in the workspace. Abort process. Prompt user to stash or commit changes first.
Missing / Empty Changelog CHANGELOG.md is missing or has no [Unreleased] items. Abort process. Inform user there is nothing to release. Prompt to run /fix-bug or manually add entries.
Tag Already Exists The calculated $NEXT_VERSION is already tagged in Git. Error state. Advise the user that the version is taken and suggest an explicit bump.
