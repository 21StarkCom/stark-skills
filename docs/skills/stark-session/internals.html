Configuration Hierarchy

Configuration objects are deeply merged via `.code-review/config.json`. Later layers override previous layers. If `session.test_command` is null, falls back to top-level `test_command`.

1. Global

~/.claude/code-review/config.json

Base developer preferences, personal defaults for PR merge strategy, devlog paths.

2. Organization

Walk parent directories from repo root.

Shared settings for a specific GitHub organization or workspace cluster (e.g., stark-skills level configs).

3. Repository

<repo_root>/.code-review/config.json

Project-specific commands (health_checks, test_command) and documentation directory definitions.

Configuration Properties Reference
Key Default Description
health_checks [] Commands to run during Phase 3 of Start Mode. Captured and reported cleanly. Non-blocking.
build_command null Command run during end routine (post-tests).
test_command null Command to run tests. Falls back to root-level config test_command if null.
doc_paths ["docs/", "CLAUDE.md"] Paths to automatically stage and commit on session end.
devlog_path null Directory to generate daily devlog entries (YYYY-MM-DD.md) on session end.
pr_merge_strategy "squash" Strategy passed to gh pr merge. Options: squash, merge, rebase.
Failure Modes & Recovery
Failure Condition System Reaction / Recovery
Missing CLAUDE.md Note in briefing, gently suggest invoking /stark-onboard-project.
gh CLI Authentication Fails Skip PR sections and external data queries; display local git state only.
Test, Build, or PR Merge Fails Report specific error to user and explicitly prompt: "Skip/Proceed anyway?"
GitHub Project API Fails / No Config Log warning internally. Skip fields section silently. Never block the pipeline.
