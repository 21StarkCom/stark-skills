stark-session

Session lifecycle orchestrator for internal contributors: context intake, git and PR state, health checks, documentation commits, project updates, and push/merge handoff.
INTERNALS • architecture • data flow • extension points

Intent: one skill, two execution modes. Start builds an operator briefing from repo, memory, config, git, project board, and health signals. End drives closeout automation with guarded test/build, optional PR merge, docs commit, project metadata updates, and push logic. The design is fail-soft: nearly every integration can degrade without blocking the rest of the session.

Architecture Notes

Config as policy layer

The skill behavior is mostly data-driven. Start mode health checks, end mode test/build commands, doc staging paths, devlog location, and merge strategy all come from the merged session config. This makes repo-level customization the main extension surface without editing the skill body.

Fail-soft orchestration

Missing config, missing memory, gh auth problems, missing project config, or project API failures do not abort execution. The skill intentionally degrades from full automation to partial automation while preserving a useful operator briefing or closeout summary.

Human checkpoint model

The skill inserts decisions only where state mutation risk is meaningful: proceeding after test/build failure, skipping failed PR merges, supplying the devlog/commit summary, and authorizing pushes for untracked branches. That keeps automation fast while still preserving operator control.

Output compression rule

Most inputs are verbose, but output is intentionally compact. Raw CLAUDE guidance, memory files, git details, and project data are distilled into a briefing or summary. This keeps the skill aligned with session management instead of turning into a document dump.

Failure posture: the skill avoids dead ends. Missing CLAUDE.md becomes an onboarding suggestion, no memory files become “no previous session context,” failing health checks stay informational, gh auth failures suppress PR operations, project config absence disables board features, and push or merge failures turn into explicit operator choices. The only behavior it refuses is unsafe automation without a checkpoint.

Start mode output

A condensed operational briefing with branch and PR posture, health signals, locally available skills, and memory-derived context so the next user turn can immediately resume work.

End mode output

A closure report describing test/build outcome, merged PRs, docs commit activity, project field adjustments, and remote push status.

Guardrails

Never dump CLAUDE.md, never use git status -uall, never commit empty staged sets, never push untracked branches without asking, and never switch to GitHub App bots for session operations.
