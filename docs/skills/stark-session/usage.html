When People Use It
Start Mode

Catch me up before coding

Use when opening a repo cold, returning after context loss, or asking what was in flight.

  • Loads CLAUDE and memory context silently
  • Shows branch, status, recent commits, stash, PRs, health results, skills
  • Ends with a concise briefing and “What are we working on?”
End Mode

Wrap the session without missing steps

Use before signing off when you need tests, PR merging, docs commits, project updates, and push handling in order.

  • Runs configured tests and build first
  • Offers PR merge actions through gh
  • Commits docs/devlog and pushes only with the right branch conditions

What You Need In Place

Minimum prerequisites

Git repository for full repo state
Optional gh CLI for PR features
Optional config at .code-review/config.json
Optional memory and CLAUDE files
Optional project config at .github/project-config.json

Common user-facing outputs

  • Start mode: a concise catch-up briefing with branch, PR, health, skills, memory, and recent activity.
  • End mode: prompts around failed checks, PR merges, commit summary text, and push decisions.
  • Final end summary: pass/fail and counts rather than long command dumps.

Failure behavior

  • No gh auth: PR sections are skipped with warning.
  • No git repo: git and PR sections are skipped.
  • Health/test/build/project API issues: reported, then flow continues.
  • No memory/config/project setup: defaults apply and the session still runs.

Guardrails

  • Never use git status -uall.
  • Do not dump full CLAUDE.md; summarize only.
  • Do not commit when nothing is staged.
  • Do not push an untracked branch without asking.

Typical Workflows

Morning Resume

Run /stark-session, read the briefing, scan the listed open PRs and failing health checks, then answer the final prompt with the next concrete task.

Ship and Sign Off

Run /stark-session end, confirm whether failed tests/build should block, merge eligible PRs, enter a one-line summary, and let the skill handle docs plus push decisions.

Low-Context Recovery

Use phrases like catch me up or what was I working on to get memory-derived context without manually reading stored notes or repo files.
