Configuration & Context

Config Hierarchy Path

Settings merge dynamically, with later paths overriding earlier ones:

  • 1. ~/.claude/code-review/config.json
  • 2. Walk parents for .code-review/config.json
  • 3. Repo root .code-review/config.json

Key Settings Session Block

  • health_checks: Commands run silently on start (Default: [])
  • test_command: Validates on end
  • build_command: Validates on end
  • pr_merge_strategy: squash/merge/rebase

Documentation Updates End Mode

On session end, stark-session commits docs securely:

  • Stages files in doc_paths (Default: docs/, CLAUDE.md)
  • Creates a daily entry in devlog_path (if set)
  • Requests a 1-line commit summary from user
Failure Gracefulness
Failure Event System Recovery / User Experience
Missing Context (No CLAUDE.md or Memory) Notes absence in briefing and suggests onboarding, but continues smoothly.
API/Auth Failures (gh auth, Project API) Skips affected PR or Project sections entirely. Shows local git state only. Never blocks.
Command Failures (Health checks, tests) Reports stderr. Tests prompt user: "Proceed anyway?". Never forcefully aborts.
Git Operations (Merge, Push fail) Reports error. Offers to skip the PR or asks how to proceed with the push.
