Primary Input

start | end

One skill, two modes. start is the default and focuses on context, git state, health checks, and a briefing.

What Users Need

Optional Config

Works with defaults, but becomes more useful when .code-review/config.json defines health checks, test/build commands, doc paths, and merge strategy.

Main Outputs

Briefing + Closeout

Start produces a concise “what changed?” briefing. End produces a session summary plus optional PR merges, docs commit, project updates, and push.

How To Invoke

Start a session

Use /stark-session or /stark-session start when you want a catch-up: recent work, branch status, open PRs, health checks, remembered context, and available skills.

End a session

Use /stark-session end when you want a clean wrap-up: run tests, optionally build, merge PRs, stage docs, write devlog notes, update project fields, and push.

Natural-language triggers

Also matches phrases like session start, session end, what was I working on, and catch me up.

Config Hierarchy

Override Order

1. ~/.claude/code-review/config.json global
2. parent directories with .code-review/config.json org
3. repo root .code-review/config.json repo

Later levels override earlier ones. The skill merges the session block across all three levels.

High-value Settings

  • health_checks: commands to run at session start
  • test_command and build_command: closeout verification
  • doc_paths: what to stage before the docs commit
  • devlog_path: enables a dated session note
  • pr_merge_strategy: squash, merge, or rebase
Key Default How it affects usage
health_checks [] Add lightweight startup confidence checks without blocking work.
build_command null Optional verification step during end.
test_command null Used for end; falls back to top-level test_command if session value is null.
doc_paths ["docs/", "CLAUDE.md"] Defines which documentation files get staged for the session update commit.
devlog_path null If set, the skill writes a dated devlog entry before committing.
pr_merge_strategy "squash" Controls how open PRs from the current branch are merged.
Common Workflows

Morning catch-up

Run /stark-session start first thing. It loads project memory, checks branch status, shows open PRs, and ends with a concise prompt to resume work immediately.

Wrap up before handing off

Run /stark-session end when you want a reliable closeout path: verify, merge, stage docs, summarize the day, update project metadata, and push if appropriate.

Mid-task context recovery

Use natural language like what was I working on or catch me up to trigger the same start-mode orientation flow without remembering the exact slash command.

Failure Handling
Condition What the skill does User impact
No CLAUDE.md Notes the gap in briefing and suggests onboarding You still get the rest of the session summary
No memory files Reports “No previous session context” No failure; only less historical context
gh auth fails Skips PR sections with a warning Git state still works; merges are not attempted
Health checks fail Shows pass/fail and error output Start mode continues normally
Tests or build fail Warns and asks whether to proceed User chooses whether closeout continues
Push fails Reports the error and asks how to proceed No hidden retry or destructive action
Project config missing Silently skips project board and field updates Core session flow still completes
What You Get Back

Start output

A compact briefing covering branch state, recent commits, PR status, health checks, memory highlights, and available skills. It is designed for quick re-entry, not for full transcript dumps.

End output

A closeout receipt covering test/build status, merged PR count, docs committed, project fields updated, and whether work was pushed upstream.

Observability

The skill tracks timing and checkpoint metrics for both modes, including per-health-check duration on start and test/build duration plus merge/doc/push counts on end.

Usage rule of thumb: choose start when you need orientation and choose end when you want the skill to turn the current branch state into a documented, merged, pushed session closeout.
