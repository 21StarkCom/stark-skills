Auth Model

User PAT (gh CLI)

  • PR merge operations
  • Push to remote
  • gh pr list / view
  • gh auth status check

unset GH_TOKEN before all session ops

Bot Token (stark-claude)

  • Project board queries only
  • github_app.py --app stark-claude token
  • Scoped: read project items
  • unset GH_TOKEN after use
Extension Points

Health Checks

Add arbitrary shell commands to session.health_checks[]. Each runs in sequence; stdout/stderr captured on failure. Non-blocking — results appear in briefing only.

Doc Paths

Extend session.doc_paths[] to auto-stage additional directories on session end. Paths are checked for existence before staging.

Devlog

Set session.devlog_path to enable automatic session devlog entries. Creates YYYY-MM-DD.md files with user-provided summary.

Project Board

Drop .github/project-config.json with project_id to enable GitHub Projects V2 integration. Surfaces in-flight/blocked items on start; updates Documentation State on end.

Observability

Follows the Skill Observability Protocol. Emits timing for each phase, per-health-check duration, test/build pass/fail, PR merge count, and push result.

Failure Recovery

Every external call has a defined fallback: skip PR sections if gh fails, continue on health check failure, ask user on test/merge/push failure. Never blocks.

Failure Modes
Failure Mode Recovery Severity
No CLAUDE.md start Note in briefing, suggest /stark-onboard-project info
No memory files start "No previous session context" — not an error info
gh auth fails both Skip PR sections, show git state only warn
Not a git repo both Skip git and PR sections entirely warn
Health check fails start Report with stderr, continue warn
Test/build fails end Warn, ask "Proceed anyway?" block
PR merge fails end Report error, offer to skip block
Push fails end Report error, ask how to proceed block
Project config missing both Skip project sections silently info
Project API call fails both Log warning, continue warn
