Extension Points

Config Hierarchy

Add new keys to the session block at any level (global/org/repo). Later levels override earlier. The merge is shallow — add new fields freely without breaking existing installs.

Health Checks

Array of arbitrary shell commands. Each runs independently, captures stdout/stderr. Add project-specific checks (DB connectivity, service health, env validation) at any config level.

Project Board Integration

Gated on .github/project-config.json. Start mode queries item statuses; end mode updates Documentation State. Extend by adding new field update rules in the end-mode logic.

Devlog System

When devlog_path is set, end mode creates daily entries (YYYY-MM-DD.md). The user provides the summary; the skill creates the file and stages it. Extend to support templates or structured formats.

Observability Protocol

Every phase emits timing metrics via the Skill Observability Protocol. Start: context_load_ms, git_state_ms, per-health-check duration. End: test/build duration, PR/doc/project counts, push result.

Failure Recovery

Every failure is non-blocking. Pattern: detect → warn → offer to continue. No phase gates on prior phase success. Extend by adding new failure handlers following the same pattern.

Auth Model

User PAT (via gh CLI)

Used for all mutating operations: PR merge, git push, issue creation. Must show as user action, not bot. unset GH_TOKEN before these operations.

Bot Token (stark-claude)

Used only for read-only project board queries in start mode. Set via export GH_TOKEN=$(github_app.py token). Always unset GH_TOKEN after query block.
