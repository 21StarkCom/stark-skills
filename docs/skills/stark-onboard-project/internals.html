Architecture Deep Dive

Idempotency & State Checks

Every phase evaluates the existing state before mutating. It ensures safe re-runs without overwriting data.

  • Git: Skips init if .git/ is valid.
  • GitHub: Skips repo creation if origin exists.
  • CODEOWNERS: Won't overwrite existing rules.
  • CLAUDE.md: Defers to stark-claude-md-improver if file exists.

GitHub App Interop (Python Script)

Integrating with org-installed GitHub Apps relies heavily on $SCRIPTS_DIR/github_app.py execution.

Validates app token access to repositories using short-lived installation tokens. Links new repos to fixed installation IDs via authenticated user REST API calls.

Project Auto-Detection Logic

Scans the working tree dynamically to determine the ecosystem, reducing manual input for standard scaffolds.

  • Node/JS: Looks for package.json, Next configs.
  • Python: Looks for pyproject.toml, requirements.txt.
  • Go/Rust/Java: Looks for standard build manifests.

Failure Modes & Recoveries

Failure Condition Node / Subsystem Recovery Strategy
GitHub repo creation fails (exists) gh cli Catch error, gracefully add remote git remote add origin ... and push upstream.
App API requires scope 403 Forbidden Inform user to run gh auth refresh -s read:user or provide exact URL for manual Org settings intervention.
No language detected Project Scan Gracefully fallback to prompting the user for Language, Framework, and Test runner.
CLAUDE.md already exists State Check Log notice and explicitly route user to /stark-claude-md-improver. Do not truncate existing instructions.

Observability & Metrics

Phase Execution Tracking

Records binary states for each phase: executed vs. skipped (due to idempotency).

GitHub App Telemetry

Tracks connected count (0-3) and logs per-app status (stark-claude, stark-codex, stark-gemini) upon skill exit.

Performance Metrics

Follows standard Observability Protocol. Captures total onboarding duration and CLAUDE.md generation origin.
