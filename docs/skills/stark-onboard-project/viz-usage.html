Common Workflows

New Project from Scratch

Navigate to an empty directory and run /stark-onboard-project. All 5 phases execute in order. You'll be prompted for repo details and to confirm the CLAUDE.md scan.

Existing Repo, Add Claude

Run /stark-onboard-project in a repo that already has git and a GitHub remote. Phases 1-2 are skipped automatically. Apps are connected and CLAUDE.md is generated.

Onboard a Different Directory

Pass the path: /stark-onboard-project ~/projects/my-service. The skill changes to that directory and runs all applicable phases.

Troubleshooting

gh not authenticated

Run gh auth login and re-run the skill. GitHub CLI must be authenticated for repo creation and app management.

App installation scope missing

Run gh auth refresh -h github.com -s read:user to add the required scope. Or add repos manually via the org settings page.

Repo already exists on GitHub

The skill handles this gracefully: it adds the remote and pushes instead of failing on gh repo create.

No language detected

If the project directory has no recognizable files, the skill asks you to specify the language and framework for the .gitignore and CLAUDE.md templates.
