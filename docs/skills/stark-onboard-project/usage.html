stark-onboard-project

Bootstrap a project from empty directory to a fully wired GitHub + Claude Code setup
USAGE VIEW
Best for first-time project setup

This skill walks an end user through the full onboarding path: initialize git, create or connect a GitHub repo in GetEvinced, grant access to the three Stark GitHub Apps, then generate the local Claude Code scaffolding. It is designed to be idempotent, so already-completed phases are skipped instead of repeated.

How To Invoke

Trigger Phrases

Use this skill when the user says things like “onboard project”, “setup claude”, “bootstrap claude”, “init project”, “create repo”, or “new project”.

onboard project setup claude create repo new project

Argument

The skill accepts an optional path. If omitted, it operates on the current working directory.

/stark-onboard-project /stark-onboard-project ~/git/my-new-app

When It Auto-Applies

It also fits naturally when work starts in a directory that has no git repository and no CLAUDE.md.

Common Workflows

Brand-New Folder

No git repo, no remote, no Claude setup. The skill runs all five phases end to end.

Local Repo Only

Git already exists, but there is no GitHub remote. The skill starts at repo creation and continues.

Repo Already On GitHub

If origin is present, it skips creation and moves to GitHub App access and Claude setup.

Claude Already Present

If CLAUDE.md exists, it will not overwrite it and instead suggests the improver skill.

GetEvinced-Only Apps

GitHub App connection is relevant for GetEvinced org repos; other orgs skip that phase.

User Decisions

Decision 1: Repo Setup

The user confirms or edits the default repo name, chooses GetEvinced or aryeh-evinced, picks visibility, and optionally adds a description.

Decision 2: CLAUDE.md Content

The skill scans the project and presents the inferred language, framework, and test runner before generating guidance.

Decision 3: Ignore .claude/

Users choose whether local Claude context remains machine-private or becomes part of a team-standardized repo workflow.

Failure Modes And Recovery

gh Not Authenticated

Recovery: run gh auth login, then re-run the skill.

Missing Installation Scope

Recovery: run gh auth refresh -h github.com -s read:user or add repositories manually through the GetEvinced org installation settings.

Existing Files

CODEOWNERS and CLAUDE.md are not overwritten. Existing work is preserved; the phase is reported as skipped.

Repo Already Exists

If GitHub reports the repo already exists, the skill falls back to adding the remote and pushing the local branch.

Why End Users Use It

Fast Start

One skill replaces a long checklist of git, GitHub, app access, and local Claude setup steps.

Safe Re-Runs

Phases are idempotent, so re-running it on a half-configured project usually finishes missing pieces instead of starting over.

Concrete Finish Line

The summary tells the user exactly what was initialized, what was skipped, and what to do next.

Best used at project inception or when inheriting a directory that is only partially wired.
