Tournament Scoring Model
Component Points Source Notes
Test Score 0 or 50 --test-command execution Binary: pass = 50, fail = 0. Skipped if no test command detected.
Correctness 1–10 Semantic evaluation (Claude judge) Does the code do what the step asked for?
Quality 1–10 Semantic evaluation (Claude judge) Clean, well-structured, follows conventions
Completeness 1–10 Semantic evaluation (Claude judge) Edge cases handled, tests written
Integration 1–10 Semantic evaluation (Claude judge) Works with existing codebase
Simplicity 1–10 Semantic evaluation (Claude judge) Simplest correct solution wins
Total 0–100 Tests (50) + Semantic (50). Ties broken by Correctness score.
Data Flow & Key Files

Input Paths

Plan file: Markdown from /stark-design-to-plan. Parsed by ## Phase / ### Task headings into ordered steps.

Inline prompt: Free-text description. Claude decomposes into 3-5 sequential steps automatically.

Scripts (Extension Points)

autopilot_dispatch.py — Creates worktrees, dispatches agents in parallel, collects diffs, runs tests. Main extension point for adding agents or changing dispatch strategy.

Prompts: global/prompts/autopilot/{agent}/implement.md — per-agent implementation preambles.

Evaluation Pipeline

Claude (orchestrator) acts as tournament judge. Reads each agent's diff, runs test command in each worktree, then scores on 5 semantic dimensions. Scores stored in tournament-log.jsonl for analysis.

Output Artifacts

~/.claude/code-review/history/autopilot/{slug}/
• steps.json — per-step results, diffs, scores
• summary.md — human-readable report
• tournament-log.jsonl — audit trail per tournament round

Failure Modes & Recovery
FailureDetectionRecovery Strategy
0/3 agents succeed All agents error or timeout Abort step, report error, ask user how to proceed
1/3 agents succeed Two agents fail Use sole survivor's output, warn: no tournament possible
All tests fail Test score = 0 for all Fall back to semantic-only scoring (max 50pts)
Diff apply failure git apply --3way fails Copy files directly from winner's worktree
Worktree creation fails git worktree add errors Fall back to sequential execution on same branch
Agent timeout Exceeds --timeout (default 900s) Disqualify timed-out agent, tournament with survivors
User Ctrl+C SIGINT caught Clean up all worktrees before exiting
Configuration & Extension Points

CLI Arguments

<plan-or-prompt> — path to plan.md or inline text (required)
--test-command CMD — override auto-detected test runner
--agents LIST — comma-separated (default: claude,codex,gemini)
--timeout N — per-agent timeout in seconds (default: 900)
--dry-run — show battle plan, don't execute

Prompt Customization

Per-agent implementation prompts live at:
global/prompts/autopilot/{agent}/implement.md

To change how an agent approaches implementation, edit its preamble. Each agent gets: preamble + prior-step context + step task + test command.

Adding a New Agent

1. Create global/prompts/autopilot/{new-agent}/implement.md
2. Add dispatch logic in autopilot_dispatch.py
3. Register in github_app.py if bot reviews needed
4. Pass via --agents claude,codex,gemini,new-agent

Observability Hooks

Task Progress

TaskCreate per phase and per step with activeForm labels. Dynamic status updates as agents complete and tournaments resolve.

Timestamped Logs

[HH:MM:SS] prefixed lines for every dispatch, agent completion (with file/line counts and duration), test results, winner declaration, and apply/commit.

Metrics Block

Final summary: total duration, steps completed, tournaments run, per-agent win rates, avg scores, test pass rates, files/lines changed, commit count.
