/stark-rename-project

Rename a project locally and on GitHub, propagate across sibling repos, reinstall symlinks

How to Invoke
/stark-rename-project <old-name> <new-name> [--dry-run]

Trigger Phrases

"rename project", "rename repo", "rename this to", or invoke directly with /stark-rename-project

Prerequisites

Clean working tree (no uncommitted changes). GitHub App admin permissions on the repo. install.sh present for symlink management.

Dry Run

Pass --dry-run to preview all changes without executing. Every phase respects the dry-run gate and prints what it would do instead.

Inputs & Outputs

Inputs

  • old-name — current project/repo name (e.g. stark-review)
  • new-name — desired new name (e.g. stark-skills)
  • --dry-run — optional flag to preview without executing
  • Names must match ^[A-Za-z0-9._-]+$

Outputs

  • Renamed GitHub repo with redirect from old URL
  • Renamed local directory
  • Updated references across project and sibling repos
  • Reinstalled symlinks pointing to new path
  • Auto-commits in sibling repos
  • Verification report with residual references

Usage Examples

Standard Rename

Rename stark-review to stark-skills
/stark-rename-project stark-review stark-skills

Renames the repo on GitHub, renames the local directory, updates all references in the project and sibling repos, reinstalls symlinks, and prints a verification report.

Preview with Dry Run

See what would change without doing it
/stark-rename-project old-name new-name --dry-run

Every phase prints what it would do instead of executing. Useful to audit before committing to the rename.

Resume After Failure

Network failure mid-rename
/stark-rename-project old-name new-name

Auto-detects partial state by comparing the remote URL to the local directory name. Resumes from the correct phase — no manual cleanup needed.

What Gets Replaced
# Pattern Scope Example

What Is NOT Changed

Protected References

  • Skill invocation names (e.g. /stark-review)
  • Skill frontmatter name: fields
  • Installed skill paths (~/.claude/skills/stark-review/)
  • GitHub App names (stark-claude, stark-codex, stark-gemini)
  • CI/CD workflow files (scanned and reported, not modified)
  • Historical document filenames
  • Binary and untracked files

Out of Scope

  • Repos outside the parent directory
  • GitHub Actions / CI pipeline updates
  • External webhooks, Slack, or Jira integrations
  • GitHub App renaming or credential changes
  • Skill command renaming
Safety Features

Idempotent

Every phase checks if its work is already done before acting. Safe to re-run after interruption.

Resumable

Detects partial completion by comparing remote URL to local dir name. Picks up from the right phase.

Literal Matching

Uses Perl \Q...\E for all replacements — no regex wildcards, no accidental matches on dots or hyphens.

Dry Run

Every phase respects --dry-run. Preview the full impact before committing to changes.

Dirty Worktree Guard

Refuses to proceed if the repo has uncommitted changes. Skips sibling repos with dirty worktrees.

Syntax Validation

Runs bash -n install.sh after modifications to catch syntax errors before they cause problems.
