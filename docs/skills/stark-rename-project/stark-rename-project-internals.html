Case-Only Rename?
If old/new differ only in case → skip local existence check, use two-step mv via temp name

Phase 2: Rename on GitHub
IDEMPOTENT DRY-RUN GATE
PATCH /repos/ORG/OLD_NAME {name: NEW_NAME} • Update git remote URLs (fetch + push) • Perl \Q..\E literal replace
Side effect: GitHub creates redirect from old URL — breaks if old name is reused later
HTTP error map: 403=no admin, 404=repo not found, 422=name taken, 5xx=retry
GitHub API Error Handling
Each HTTP status → specific actionable error message • Verify response name matches NEW_NAME
Recovery: Resumable state detection (Phase 1c) allows re-run after transient failures
Critical: All subsequent paths must use $PARENT/$NEW_NAME — cwd is invalidated by mv
Phase 3a: Local Directory Rename
IDEMPOTENT DRY-RUN GATE
mv $PARENT/$OLD → $PARENT/$NEW • cd into new path immediately • Case-only: two-step via temp
Idempotency: Skip if NEW exists and OLD does not
Order matters: Must uninstall BEFORE Phase 4 modifies install.sh targets
Phase 3b: Uninstall Old Symlinks
install.sh --uninstall (preferred) • Fallback: find all symlinks under ~/.claude + ~/git/Evinced/.code-review resolving to old path
Safety: Uses python3 os.path.realpath (not readlink -f — unavailable on macOS) • Only deletes symlinks, never data
File discovery: git grep -Il "" (tracked text files only, skips binary)
Phase 4: Update References (Self)
DRY-RUN GATE
5 replacement patterns (most-specific-first) • Perl \Q..\E literal matching • Exclusion rules for skill identity
Validation: bash -n install.sh after modifications • Track every modified file
Discovery: All dirs under $PARENT with .git/ whose origin matches same HOST + ORG
Phase 5: Update References (Siblings)
DRY-RUN GATE
Pre-flight clean worktree check per repo • Same 5 patterns + exclusions as Phase 4 • Auto-commit per repo
Safety: git add specific files (not -am) • Skip repos with uncommitted changes • Array quoting for paths
Depends on: Phase 3b (uninstall) + Phase 4 (updated install.sh)
Phase 6: Reinstall Symlinks
DRY-RUN GATE
cd $PARENT/$NEW_NAME • ./install.sh • Report actionable error if install fails
Extension point: install.sh is the single source of truth for symlink targets
Checks: Remote connectivity, stale symlinks, residual references, CI/CD files
Phase 7: Verification
git ls-remote origin • Scan ~/.claude for stale symlinks • Grep all 5 patterns with exclusion rules • Report .github/workflows/ matches
False-positive filter: Same exclusion rules as Phase 4 applied to residual scan
Observability: Follows Skill Observability Protocol for timing + metrics
Phase 8: Summary Report
Files changed per repo • Pass/fail verification • Residual refs • CI/CD warnings • Skipped siblings • cd command
Metrics: API duration, file counts, symlinks removed/created, checks passed/failed

Resumable State Machine

State A: Remote=new, Local=old

GitHub rename succeeded but local rename did not. Resume from Phase 3 (local directory rename). The remote URL already points to the new name.

State B: Remote=old, Local=new

Local rename happened but GitHub rename did not. Resume from Phase 2 (GitHub API rename). This is unusual but possible if the skill was interrupted between directory operations.

State C: Both=new

Both renames completed but symlinks may be stale. Resume from Phase 3b (symlink cleanup + reinstall). Fast path for interrupted post-rename cleanup.

No Match: Fresh Run

Neither resume state detected. Execute the full pipeline from Phase 1d (standard validation) through Phase 8 (summary).

Extension Points

Adding Replacement Patterns

New patterns go in Phase 4a (deterministic) or 4b (heuristic). Deterministic patterns are applied to all tracked text files; heuristic patterns need a file-type restriction. Always add most-specific patterns first to prevent double-replacement.

Adding Exclusion Rules

New exclusions go in Phase 4c. Must be applied consistently in Phase 4, Phase 5, and Phase 7 (verification). If a pattern should be excluded only in certain file types, document the scope explicitly.

Sibling Repo Discovery

Phase 5a discovers siblings by scanning $PARENT for .git/ directories matching the same HOST + ORG. To support additional discovery strategies (e.g., GitHub API for all org repos), extend the discovery loop without changing the per-repo replacement logic.

Symlink Install Locations

Phase 3b fallback scans ~/.claude and ~/git/Evinced/.code-review. Additional install destinations require adding paths to the fallback scan list. The install.sh --uninstall path handles this automatically.

Known Pitfalls & Mitigations
Pitfall Impact Mitigation
Using \b word boundaries Hyphens/dots are word boundaries — matches inside longer names Custom lookaround: (?<![A-Za-z0-9._-])..(?![A-Za-z0-9._-])
Using sed for replacements . is a regex wildcard — foo.bar matches fooXbar Perl \Q..\E for literal string matching
Using readlink -f Not available on macOS python3 -c "import os; print(os.path.realpath(...))"
Modifying files before uninstalling symlinks install.sh targets change, --uninstall can't find old paths Strict ordering: uninstall (3b) before modify (4)
Forgetting to cd after mv All subsequent commands operate from invalid cwd Immediate cd $PARENT/$NEW_NAME after mv
git commit -am in siblings Sweeps unrelated changes into rename commit git add specific files — array quoting: "${files[@]}"
Hardcoding GetEvinced or github.com Breaks for other orgs/hosts All values parsed from git remote get-url origin
