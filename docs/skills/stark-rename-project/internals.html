Skill Overview

Renames a project both locally and on GitHub, propagates the name change across sibling repositories under the same parent directory, and cleanly handles symlink recreation. Built with resumability, idempotency, and robust heuristic regex patterns to prevent accidental replacements.

Deep Dive: Data Patterns & Regex

Deterministic Patterns

Uses Perl literal matching \Q...\E to ensure . isn't treated as a regex wildcard.

$PARENT/$OLD_NAME $ORG/$OLD_NAME $HOST:$ORG/$OLD_NAME $HOST/$ORG/$OLD_NAME

Heuristic Boundaries

Matches bare $OLD_NAME using custom lookarounds to avoid substring matches (e.g., matching inside stark-review-improvement):

(?<![A-Za-z0-9._-])OLD_NAME(?![A-Za-z0-9._-])

Restricted strictly to *.md, *.json, *.sh files.

Exclusion Rules

Patterns explicitly excluded from automated replacement:

  • Skill invocation prefixes: /{old-name}
  • Skill identities: name: '{old-name}'
  • Installed skill paths: ~/.claude/skills/...
  • Historical doc filenames.
  • Contents of .git/ or .github/workflows/
