Key Operational Highlights

Idempotency & Auto-Resume

If a rename fails midway, re-running the skill will pick up where it left off. It detects the current state by comparing the local directory name against the GitHub remote origin name.

Phase 2 Resume Phase 3 Resume

Safe Dry-Run Mode

Pass the --dry-run flag to simulate the exact changes. It will list API requests, directories affected, references that would change across sibling repos, and old symlinks to be pruned.

Exclusion Rules

The skill intelligently avoids renaming items that share the project's name but serve different functions:

  • Skill commands (e.g., /{old-name})
  • Historical markdown filenames
  • Installed paths under ~/.claude/
  • GitHub Apps & Frontmatter definitions

Manual Post-Task Actions

Some external systems cannot be automatically updated. Review the final output for warnings about:

  • .github/workflows/*.yml files
  • External webhooks or Jira links
  • Repos outside the current parent folder

Common Regex & Replacement Safety

Pattern / Approach Why it's used Risk Prevented
Perl Literal Matching (\Q...\E) Safely matches exact repo URLs. Prevents sed from interpreting periods (e.g. foo.bar) as wildcards.
Custom Lookarounds ((?<![...])...(?![...])) Boundary checks for repo names with hyphens. Prevents matching substrings (e.g., matching stark-review inside stark-review-plan). Standard \b breaks on hyphens.
Uninstall Symlinks Before Code Mod Preserves the original target mapping needed by install.sh --uninstall. Prevents orphaned symlinks that point nowhere after install.sh is updated with the new name.
Git Arrays ("${files[@]}") Properly quotes spaces when auto-committing siblings. Prevents shell injection and crashes on repos with spaces in their filenames.
