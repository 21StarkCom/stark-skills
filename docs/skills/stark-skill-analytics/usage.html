Skill Visualization

stark-skill-analytics

Usage-focused map of how to invoke the skill, what it reads, what it computes, and what it returns.
Audience: usage

This skill turns Claude Code history into an analytics report for skill adoption, quality, project affinity, and workflow sequences. It is best used when you want to answer: which skills get used, where, how often, and whether they appear healthy.

Primary Input

Slash-command history from ~/.claude/history.jsonl, streamed line by line to find actual skill invocations.

Optional Enrichment

Run history under ~/.claude/code-review/history, benchmark data, and the registered skills list from CLAUDE.md.

Main Output

A markdown report with rankings, trends, sequences, quality signals, discovery gaps, and concrete recommendations.

How To Invoke

Quick Start

Use the skill with no arguments when you want the broadest picture across every skill seen in Claude Code history.

/stark-skill-analytics

Single Skill Deep Dive

Focus on one skill to inspect project spread, argument patterns, monthly usage, and quality signals for that command alone.

/stark-skill-analytics --skill stark-review

Ranking View Only

Use table mode when you only need the leaderboard and trend scan, without long per-skill sections.

/stark-skill-analytics --format table

Typical Return

The skill writes a markdown report, prints it to the terminal, and includes metrics like total invocations, sessions analyzed, history files, and recommendations generated.

Arguments
Argument What It Changes Good For
--skill <name> Generates a deep-dive report for one named skill. Investigating adoption and quality of a single workflow such as stark-review.
--format table Outputs only the usage rankings table and skips detailed sections. Fast triage, dashboards, or quick status checks.
--format full Returns the full detailed report. This is the default behavior. Portfolio reviews, cleanup planning, and adoption analysis.
No arguments Runs across all skills with the full report format. Regular reporting and broad discovery work.

What Users Learn From The Report

Usage Rankings

See which slash commands dominate usage, how many projects they appear in, and whether each skill is rising, stable, declining, or newly adopted.

Adoption Timeline

Understand when a skill first appeared and how its growth evolved over time, grouped by month.

Project Affinity

Spot which repositories use only a narrow subset of available skills, and which ones have heavy multi-skill workflows.

Sequence Detection

Find the most common back-to-back skill pairs within a session, which is useful for workflow consolidation.

Quality Signals

Cross usage volume with failures, timeouts, empty-result patterns, duration, and benchmark trends where data exists.

Recommendations

The report explicitly recommends which skills to remove, promote, fix, investigate, or combine into a new workflow skill.

Common Workflows

Portfolio Health Scan

Run with no arguments to get the full report, then use the Discovery and Recommendations sections to identify dead skills, emerging skills, and missing quality coverage.

Investigate One Skill

Run with --skill <name> when a team asks why one workflow is popular, declining, or problematic.

Quick Weekly Snapshot

Use --format table for a compact leaderboard of invocation counts, project spread, and trend direction.

Failure Handling
Situation Observed Behavior User Meaning
history.jsonl missing Hard error and abort. No reliable session history exists, so the skill cannot infer usage.
history.jsonl empty Hard error: no session history found. There is nothing to analyze yet.
No slash-command lines Returns a report stating no skill invocations were found. Claude Code was used, but not through slash skills.
Quality history missing Logs a warning and continues with usage-only reporting. You still get adoption analytics, but health metrics will be thin.
Bad JSON in history files Skips corrupt lines/files and increments warning counts. The report remains usable, but partial data may lower confidence.
Unknown --skill name Returns a not-found message with available skill names. The requested skill has never been seen in usage history.
Practical takeaway: this is a reporting skill, not an execution skill. It reads historical artifacts, correlates them, and emits markdown insights. If you need agent-level review performance or finding-level analysis rather than skill adoption, the skill itself points you to /stark-metrics.
