/stark-skill-analytics

Analyze skill usage patterns and quality metrics across all Claude Code sessions
AUDIENCE: USAGE
Quick Start
/stark-skill-analytics

No arguments required. Produces a full analytics report across all skills and projects.

Invocation Options
Flag Description Example
--skill NAME Deep-dive report for a single skill --skill stark-review
--format table Summary rankings table only (no per-skill details) --format table
--format full Full report with per-skill breakdown (default) --format full
(no args) Full report across all skills /stark-skill-analytics
# Deep-dive into a specific skill
/stark-skill-analytics --skill stark-review
# Quick summary table only
/stark-skill-analytics --format table
Data Sources
1
Session history — ~/.claude/history.jsonl
Every slash-command invocation with timestamp, project, and session ID
2
Run history — ~/.claude/code-review/history/
Per-run JSON files with duration, status, output counts, and agent results
3
Benchmarks — ~/.claude/code-review/scripts/benchmarks/
Codex benchmark pass/fail rates and performance trends
4
CLAUDE.md — current repo
Registered skill list for discovery analysis (used vs. unregistered)
What You Get

Usage Rankings

All skills ranked by invocation count with project spread, first/last used dates, and 30-day trend (rising, stable, declining, new).

Adoption Timeline

Chronological view of when each skill was first adopted, grouped by month. See how your skill repertoire grew over time.

Quality Signals

Cross-referenced usage + run data: timeout rates, failure rates, empty results, and quality degradation trends per skill.

Skill Sequences

Top 10 most common consecutive skill pairs within sessions. Reveals natural workflows like /stark-review followed by /stark-review-improvement.

Discovery

Skills used but not registered in CLAUDE.md, and registered skills never invoked. Keep your skill catalog accurate.

Recommendations

Actionable advice: remove dead skills, fix high-error skills, promote underused gems, and combine frequent sequences into workflows.
