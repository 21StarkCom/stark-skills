Argument Surface
CLI Shape: [--skill <name>] [--format table|full]

`--skill <name>`

Narrows aggregation, sequence extraction, argument analysis, and output filename to one skill. Also changes report layout to a deep-dive format.

`--format table`

Short-circuits detailed per-skill sections and emits summary rankings only. Intended for quick terminal inspection or low-noise dashboards.

`--format full`

Default mode. Builds rankings, adoption timeline, detailed per-skill sections, quality signals, discovery gaps, project affinity, and recommendations.

No Arguments

Full-fleet report across all discovered slash-command skills, including registered-but-unused and used-but-unregistered comparisons.

Extension Points

New Data Sources

Add another external input by following the Phase 2 pattern: scan source, tolerate parse failures, normalize to a per-skill schema, then merge into the Phase 3 cross-reference layer.

Trend Heuristics

Trend status is a threshold classifier over the last 30 days vs the prior 30 days. To tune sensitivity, change the comparison windows or the ±20% boundary without disturbing downstream report shape.

Recommendation Engine

The most leverage sits here: dead-skill removal, underused skill promotion, quality regressions, and sequence-combination ideas all come from composable predicates over usage and quality stats.

Skill-Specific Output Summaries

Phase 2 already branches by history subtype such as `extract-docs`, `plan-to-tasks`, and `reviews`. New skill families can register custom output-summary logic while still contributing to shared metrics.

Report Shapes

The markdown generator has three natural seams: full report, summary table, and single-skill deep dive. Additional renderers such as CSV or JSON could be added after Phase 3 with minimal disruption.

Observability Protocol

The skill is required to emit timestamped phase logs and a terminal metrics block. Any modification that adds new subphases should preserve that instrumentation boundary for operational debugging.

Failure Handling Model
Failure Mode Severity Behavior Why It Matters Internally
`history.jsonl` missing fatal Abort with explicit error Phase 1 is the root fact table; nothing meaningful can be inferred without invocation history.
`history.jsonl` empty fatal Abort with “No session history found” Distinguishes missing source from a present-but-empty environment.
No skill invocations found soft Emit empty-state report Preserves report pipeline and diagnostics for a clean environment or onboarding repo.
History directory missing degraded Skip quality metrics and continue Maintains adoption analytics even when run telemetry is not installed.
Corrupt JSON history files degraded Increment warning count and continue Prevents one bad artifact from suppressing the global report.
Unknown `--skill` value fatal Return not-found message with available skills Avoids silently generating an empty single-skill report.
Insights dir not writable degraded Print report only and warn on save failure Keeps terminal deliverability even when persistence fails.
Architecture Notes

Why Streaming Matters

The implementation note explicitly forbids loading large `history.jsonl` files wholesale. That keeps memory bounded and makes this skill safe to run against long-lived local histories.

Identity Resolution

The skill uses slash-command names as the common join key across prompt history, run history subdirectories, and registered inventory. Any rename policy should consider backward-compatible aliases.

Project Affinity Layer

Project paths are shortened for display but should remain stable enough to compare repository-level adoption. This is the best place to introduce org/team rollups later.

Sequence Mining

Consecutive pair counting is intentionally simple and session-local. A future extension could add n-gram workflows or latency-aware sequence confidence without changing the base report contract.
