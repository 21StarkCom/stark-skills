Analysis Engine Details

1. Extracted Entities

Data points synthesized across sources:

  • Skill Signature: Name, arguments, invocation type.
  • Temporal: first_used, last_used, 30d trend (rising, stable, declining, new).
  • Quality: Success rate, timeout rate, avg duration.
  • Session Context: Skill chaining (A -> B).

2. Recommendation Heuristics

The engine generates automated advice based on triggers:

  • Remove: In CLAUDE.md, but 0 runs in last 60 days.
  • Fix: Timeout rate > 20% or Failure rate > 10%.
  • Promote: Underused but 100% success rate across few projects.
  • Sequence: High occurrence of contiguous skill invocations.

3. Technical Constraints

Performance and boundary limits:

  • Streams history.jsonl line-by-line (O(N) memory).
  • Uses inline Python via bash for aggregation (no external `.py` scripts).
  • Does NOT invoke external APIs or modify configs.
  • Excludes plain prompt text tracking.

Failure Modes & Fallbacks

Failure Condition System Response / Recovery
history.jsonl doesn't exist or is empty Abort Error: "No history.jsonl found." Claude Code session history is strictly required.
history/ dir doesn't exist Graceful Downgrade Skip quality metrics, generate report using usage volume data only.
Corrupt JSON in history files / jsonl Skip Line/File Skip malformed entry, increment warning count, continue processing.
Target --skill <name> not found Abort Print: "Skill not found in history. Available: {list}"
insights/skills/ dir not writable Fallback Print full report to terminal, warn about disk save failure.
