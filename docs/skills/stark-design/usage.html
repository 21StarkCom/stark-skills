Quick Start
# From an inline prompt
> /stark-design "Add a webhook notification system for build events"

# From a requirements file
> /stark-design docs/requirements/webhooks.md

# With options
> /stark-design docs/requirements/webhooks.md --agents claude,gemini --output docs/specs/webhooks-design.md

What You Get

Design Document

Synthesized spec at docs/specs/YYYY-MM-DD-<topic>-design.md. Combines the strongest elements from all 3 agent designs, with cross-review weaknesses addressed.

Review Summary

Written alongside the design as <name>.design-review.md. Contains the full scorecard, per-agent strengths/weaknesses, and synthesis decisions.

PR Comment

If a PR is detected, a summary is automatically posted via stark-claude bot. Includes the scorecard and a link to the output file.

Run History

Full artifacts saved to ~/.claude/code-review/history/prompt-to-design/<slug>/ — generation JSON, cross-review JSON, synthesis, and summary.

Common Patterns

Quick Feature Design

Describe the feature inline. Best for small-to-medium features where you don't have a formal requirements doc yet.

/stark-design "Add rate limiting to the public API with per-tenant quotas"

From Requirements File

Point to a detailed requirements doc for larger features. The full file content is sent to each agent.

/stark-design docs/requirements/auth-rewrite.md

Preview Before Committing

Use --dry-run to see the designs and scorecard without writing any files. Useful to validate the approach before persisting.

/stark-design "migrate to event sourcing" --dry-run

Full Pipeline

Chain the entire workflow: design → review → plan → review → tasks → execute. Each step feeds the next.

/stark-design "new billing system"
/stark-review-design docs/specs/...-design.md
/stark-design-to-plan docs/specs/...-design.md

Failure Recovery
FailureWhat Happens
No input Prompts you: "What should the design cover?"
File not found Searches docs/ for partial name matches
0/3 agents succeed Aborts with dispatch diagnostics
1/3 agents succeed Uses single design, skips cross-review, warns
2/3 agents succeed Cross-reviews with 2 designs (4 reviews instead of 6)
Scripts missing Prompts: "Run install.sh to set up stark-skills"
