stark-review-plan

Multi-agent design document review with autonomous fix loops, final validation, and persisted review artifacts.
USAGE VIEW • how to invoke, what happens, what you get back

When To Use It

Use this skill when you want a plan or spec stress-tested before implementation. It dispatches 21 reviewers in parallel: 3 LLMs across 7 review domains, classifies real issues vs noise, applies fixes for up to N rounds, then runs a final review-only pass.

"review this plan" "review this spec" "review design doc" /stark-review-plan
/stark-review-plan <path> [--rounds N] [--dry-run] [--force]

What You Provide

  • <path> to a markdown plan or design doc
  • Optional --rounds N to cap fix cycles
  • Optional --dry-run for review only
  • Optional --force to bypass dirty-file protection

What You Get

  • Terminal summary with issue/noise counts
  • .review.md file beside the plan
  • Round history and metrics
  • Optional PR comments per agent and orchestrator

Inputs And Outputs

Inputs

Markdown plan path, optional round limit, and execution-mode flags. The skill also inspects git state, PR context, local CLIs, and internal review config.

Decision Logic

It distinguishes real issues from false positives, tracks recurring findings across rounds, and treats zero successful agents as a dispatch failure instead of a clean review.

Outputs

Consolidated review summary, per-round history, change diff, optional per-agent PR comments, and observability metrics that make long runs understandable while they are still in progress.

Common Workflows

1. Tighten A New Spec Before Implementation

  1. Run the skill on the draft markdown plan.
  2. Let the loop fix missing edge cases, rollback gaps, and contradictions.
  3. Read the final unresolved section to decide what still needs human judgment.

2. Compare Review Signal Without Editing

  1. Run with --dry-run.
  2. Inspect issue vs noise counts and misalignment analysis.
  3. Use the result to decide whether prompt tuning or plan edits are more valuable.

3. Review A Plan Already Attached To A PR

  1. Run on a branch with an open PR.
  2. The skill authenticates a GitHub app if possible.
  3. Each agent posts raw findings, then the orchestrator posts the classified summary.

4. Diagnose Broken Dispatch

  1. If all sub-agents fail, the run switches to diagnostics.
  2. CLI availability and a single-agent probe are executed.
  3. The summary recommends likely fixes such as auth, installation, or network remediation.
