Scale: 3 LLMs × 7 domains = 21 sub-agents
Interface: <path> [--rounds N] [--dry-run] [--force]
Artifacts: summary, review file, PR comments, history

stark-review-plan is an orchestration skill for reviewing plans and specs as operational systems rather than static text. It validates a target markdown file, fans out parallel domain reviews across Claude, Codex, and Gemini, classifies the returned findings into fixable signal versus noise, edits the plan across bounded fix rounds, then performs a final review-only pass before persisting structured history and optionally posting every agent’s raw output back to GitHub.

Internal emphasis: dispatch mechanics, classification rules, observability, persistence, and prompt-tuning feedback loops.

Key Behaviors

Failure-Aware Cleanliness

A round with zero successful sub-agents is explicitly treated as infrastructure failure, not plan quality. The skill runs CLI diagnostics and avoids wasting time on a final re-dispatch that would fail identically.

Signal Separation

Headline counts exclude noise. Fixes and recurring issues represent real defects, while false positives and stylistic noise are preserved for learning but not inflated into issue totals.

Cross-Agent Confidence

The same concern raised by multiple models against the same section increases confidence. This reduces overfitting to one model’s quirks while still allowing minority findings to be tracked as noise or ignored signal.

Autonomous Improvement Loop

The skill edits the plan directly between rounds, then evaluates whether residual issues remain. This makes the workflow a bounded repair system rather than a one-shot static reviewer.

Observability Contract

Task-based progress, timestamped phase transitions, 5-minute checkpoints, end-of-run metrics, and improvement flags are mandatory outputs. Long-running reviews are expected to be inspectable while in progress.

GitHub As Durable Memory

Per-agent raw findings are posted under each bot identity when a PR exists. This preserves reviewer-specific output for later prompt tuning and failure analysis beyond the consolidated orchestrator summary.
