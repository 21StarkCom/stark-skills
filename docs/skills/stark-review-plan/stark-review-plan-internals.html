Output Artifacts

5a. Terminal Summary

Headline counts (issues vs noise vs ignored), signal-to-noise ratio, full findings table, fixed/recurring/unresolved sections, misalignment analysis, diff of changes, prompt improvement assessment, and metrics block with timing and improvement flags.

5b. Review File

{plan-name}.review.md alongside the plan. Skipped in --dry-run. Contains the same consolidated summary as the terminal output.

5c. PR Comments (3 bots)

Each agent posts raw findings under its own GitHub App identity (stark-claude, stark-codex, stark-gemini). Then stark-claude posts the orchestrator's classified summary. Zero-finding agents still post a status comment.

5d. History

~/.claude/code-review/history/plan-reviews/{name}/rounds.json (all round data) and summary.md. Removes in-progress.json on completion.

Config & Extension Points

Config Hierarchy

Global: ~/.claude/code-review/config.json
Org: ~/git/Evinced/.code-review/config.json
Repo: {repo}/.code-review/config.json

Key fields: plan_review.max_rounds, plan_review.fix_threshold, disabled_domains

Prompt Override Layers

Global prompts: global/prompts/plan-review/{agent}/{domain}.md
Repo overrides: {repo}/.code-review/plan-prompts/{agent}/{domain}.md

Repo-level prompts fully replace the global prompt for that agent+domain pair.

Failure Handling

Dispatch failure (0 succeeded): diagnostics run, error template used, Phase 3 skipped.
Low coverage (<50%): warning emitted, review continues.
Auth failure: PR posting skipped, review still runs.
Crash recovery: in-progress.json enables resume.

Observability System

Task-Based Progress

5 top-level tasks (one per phase) with child tasks per round in Phase 2. Each set to in_progress before starting, completed when done. Powers the CLI spinner.

Timestamped Logs

T0 recorded at start. Every phase transition and key event logged with [HH:MM:SS] prefix. Includes sub-agent success counts, finding breakdowns, and fix summaries.

Metrics Block

Printed at end: total duration, per-phase timing, per-round breakdown, issue/noise counts, agent success/fail, round count. Plus improvement flags for bottlenecks.
