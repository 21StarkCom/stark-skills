/stark-init-docs

Scaffold standardized developer documentation into any repository
AUDIENCE: USAGE
Quick Start
/stark-init-docs --template

Invoke with no arguments to see an interactive mode picker, or pass flags directly.

Four Modes
--template
Empty skeleton with standard dirs and config files
--backfill
Generate docs from git history and codebase analysis
--upgrade
Migrate existing scattered Markdown into standard layout
--clean
Remove scaffold files, preserve user content

Modes are combinable: --upgrade --backfill migrates existing docs then generates new ones

Common Usage Patterns

New Project

Start with /stark-init-docs --template to get the skeleton, then fill in docs manually as you build features.

/stark-init-docs --template

Existing Project, No Docs

Use --backfill to auto-generate documentation from your git history, PRs, and codebase structure.

/stark-init-docs --backfill

Existing Project, Scattered Docs

Combine --upgrade --backfill to migrate existing .md files into the standard layout, then fill gaps with generated content.

/stark-init-docs --upgrade --backfill

Remove Scaffold

Use --clean to strip skeleton files while preserving any real docs your team has written.

/stark-init-docs --clean

Error Handling

Templates Not Found

If ~/.claude/code-review/standards/templates/ is missing, run install.sh in stark-skills first.

Not a Git Repo

Must be run inside an initialized git repository. Run git init first.

gh CLI Unavailable

--backfill continues without PR data. Warns and generates from commits + codebase only.
