Mode Matrix

Template

Creates docs/{specs,plans,adr,guides,reference,architecture}, copies standard templates, fills repo-name placeholders, generates CODEOWNERS if absent, then commits only if files changed.

Backfill

Runs template first, mines history and repo config, generates conservative ADRs, stub specs from significant merged PRs, developer guides from actual commands, then rewires mkdocs.yml.

Upgrade

Finds Markdown outside docs/, classifies it into spec/adr/guide/reference/architecture, moves files with git mv, repairs internal links, updates nav, then fills structural gaps via template.

Clean

Requires explicit confirmation, removes only scaffold artifacts and empty directories, preserves user-generated docs and meaningful history, then skips commit if there is nothing to remove.

Internals View

Control Plane

The skill is effectively a mode dispatcher wrapped around a shared mutation engine. The engine enforces idempotency, preserves existing user content, and treats git state as the source of truth for both safety and commit creation.

Decision Boundaries

There are three hard gates: missing repo/templates cause immediate failure, no-arg invocation pauses for user selection, and clean mode pauses for explicit confirmation before destructive scaffold removal.

Extension Points

Add new analyzers in the backfill file-scan stage, extend the document classifier signal table, or broaden mkdocs.yml nav generation. The safest customization surface is rule tables, not ad hoc file mutation logic.

Safety Invariants

Never overwrite existing scaffold files during template creation
Never delete user-generated docs during clean
Skip empty commits across all modes
Continue backfill without PR data if gh is unavailable
Operational Risks

Template Source Missing

The skill is tightly coupled to a specific template directory. If that path is absent, execution halts immediately. This is the highest-friction dependency because every generation path eventually relies on scaffold assets.

Classification Drift

Upgrade mode uses heuristic content signals. Misclassification falls back to reference, which is safe for preservation but can weaken doc discoverability until manually corrected.

History Quality Dependency

Backfill output quality depends on commit hygiene, PR titles, and discoverable config files. The skill is conservative by design, so weak repository metadata yields fewer ADRs and thinner generated specs.

Contributor Summary

Primary Responsibility

Normalize documentation structure across repositories while preserving existing content and fitting within git-native workflows.

Best Internal Abstraction

Treat each mode as a policy layer over one shared engine: inspect, decide, mutate, wire nav, commit, report metrics.

Most Important Rule

Idempotency over aggression

Existing files are skipped, meaningful docs are preserved, and destructive work requires confirmation.
