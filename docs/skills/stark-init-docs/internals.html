\n Medium complexity · ~325 lines\n

\n This skill is an orchestration layer over repository introspection, template materialization,\n content synthesis, migration, and safe cleanup. Its core design principle is idempotent\n filesystem mutation: each mode builds or rewrites documentation state without guessing beyond\n evidence from git history, repo config, and explicit user confirmation.\n

Internal Architecture Notes
\n
\n
\n

Data Flow Contract

\n

\n Evidence flows from filesystem and VCS into a narrow generation layer. The skill does not invent\n architecture: it extracts repository facts, applies classification heuristics, then writes\n documentation artifacts and navigation metadata.\n

\n
\n
\n

Idempotent Mutation Strategy

\n

\n Every branch uses skip semantics or guarded deletion. Existing destinations short-circuit writes,\n commits are omitted when empty, and cleanup removes only scaffold-owned files or empty directories.\n

\n
\n
\n

Decision Surfaces

\n

\n The main decision points are mode selection, document classification during upgrade, technology\n significance during ADR generation, and cleanup authorization. Each one narrows potentially risky mutations.\n

\n
\n
\n

Artifact Topology

\n

\n The skill centralizes documentation into docs/specs, docs/plans,\n docs/adr, docs/guides, docs/reference,\n and docs/architecture, with mkdocs.yml as the indexable nav root.\n

\n
\n
\n

Extension Points

\n

\n Internals can extend template sources, add new manifest analyzers, improve PR significance scoring,\n expand markdown classification rules, or enrich nav generation without changing the mode contract.\n

\n
\n
\n

Failure Semantics

\n

\n Template absence and non-git execution are hard failures. Missing GitHub CLI is a soft failure.\n File move conflicts during upgrade are warnings. The design prefers degraded output over aborted runs\n when safe continuation is possible.\n

\n
\n
\n\n
Extension Map
\n
\n
\n

Analyzer Plug-ins

\n

\n Add parsers for new ecosystems such as Bazel, Terraform, Helm, or monorepo workspace files to deepen\n backfill accuracy while preserving the conservative ADR policy.\n

\n
\n
\n

Classification Heuristics

\n

\n Upgrade mode can evolve from keyword matching into richer structural parsing, frontmatter support,\n or ML-assisted classification as long as uncertain cases still default to reference.\n

\n
\n
\n

Ownership + Provenance

\n

\n Cleanup safety improves if generated files carry explicit markers or frontmatter signatures. That reduces\n ambiguity around whether a CODEOWNERS or config file originated from this skill.\n
