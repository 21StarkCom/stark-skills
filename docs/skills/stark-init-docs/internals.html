stark-init-docs

Repo documentation scaffold and migration skill
INTERNALS · architecture · data flow · extension points
Medium complexity · ~325 lines

This skill is an orchestration layer over repository introspection, template materialization, content synthesis, migration, and safe cleanup. Its core design principle is idempotent filesystem mutation: each mode builds or rewrites documentation state without guessing beyond evidence from git history, repo config, and explicit user confirmation.

Internal Architecture Notes

Data Flow Contract

Evidence flows from filesystem and VCS into a narrow generation layer. The skill does not invent architecture: it extracts repository facts, applies classification heuristics, then writes documentation artifacts and navigation metadata.

Idempotent Mutation Strategy

Every branch uses skip semantics or guarded deletion. Existing destinations short-circuit writes, commits are omitted when empty, and cleanup removes only scaffold-owned files or empty directories.

Decision Surfaces

The main decision points are mode selection, document classification during upgrade, technology significance during ADR generation, and cleanup authorization. Each one narrows potentially risky mutations.

Artifact Topology

The skill centralizes documentation into docs/specs, docs/plans, docs/adr, docs/guides, docs/reference, and docs/architecture, with mkdocs.yml as the indexable nav root.

Extension Points

Internals can extend template sources, add new manifest analyzers, improve PR significance scoring, expand markdown classification rules, or enrich nav generation without changing the mode contract.

Failure Semantics

Template absence and non-git execution are hard failures. Missing GitHub CLI is a soft failure. File move conflicts during upgrade are warnings. The design prefers degraded output over aborted runs when safe continuation is possible.

Extension Map

Analyzer Plug-ins

Add parsers for new ecosystems such as Bazel, Terraform, Helm, or monorepo workspace files to deepen backfill accuracy while preserving the conservative ADR policy.

Classification Heuristics

Upgrade mode can evolve from keyword matching into richer structural parsing, frontmatter support, or ML-assisted classification as long as uncertain cases still default to reference.

Ownership + Provenance

Cleanup safety improves if generated files carry explicit markers or frontmatter signatures. That reduces ambiguity around whether a CODEOWNERS or config file originated from this skill.
