This skill is an orchestration layer, not just a prompt. It validates a design document, dispatches independent planning agents, cross-reviews every plan, scores the field, then synthesizes a final plan with rollback, verification, and history artifacts.

Control Model

Claude Code acts as orchestrator. Agents only generate plans and reviews; synthesis remains local so policy, scoring interpretation, and final artifact shape stay deterministic.

Decision Model

Cross-review averages create a winner or tie. Weakness consensus removes bad ideas. Tight score gaps add a caution flag for synthesis review.

Artifact Contract

Outputs include the plan markdown, review summary, raw JSON dispatch traces, synthesis summary, optional PR comment, and cleanup of temporary review state.

Why the architecture matters

  • Independent generation reduces anchoring bias across agents.
  • Cross-review creates structured disagreement before synthesis.
  • Winner-plus-merge avoids averaging everything into vague prose.
  • History artifacts make the run inspectable and replayable.

Dimension Why it matters in synthesis Merge implication
Completeness Detects missing design sections or omitted delivery work. Pull missing task coverage from a stronger non-winning plan.

Winner rule

Highest average score becomes the base plan. If top-two are within 0.5 points, the orchestrator treats synthesis as a tie and blends both more evenly.

Weakness consensus rule

If both reviewers of a plan identify the same flaw, that element is excluded from synthesis instead of being softened or rationalized.

Improvement flags

Slow agents, parse failures, close races, or winning-plan low scores are explicitly surfaced so maintainers know where to inspect the output manually.

Failure And Degraded Paths

Input failures

No path, unreadable file, or no candidate match stop the run before any expensive dispatch. Dirty design files require either cleanup or --force.

Generation failures

0/3 successes abort. 1/3 uses a single surviving plan without cross-review. 2/3 still cross-review the available pair using four review edges instead of six.

Review failures

Cross-review parse failures fall back to raw outputs. If every review parse fails, synthesis uses structural quality signals like coverage and line count.

Operational dependencies

Missing scripts imply install/setup failure. GitHub auth failure degrades gracefully by skipping PR posting rather than blocking plan generation.

Observability Contract

Task lifecycle

Every phase must create a user-visible task with an active-form label, making the pipeline legible during long-running agent work.

Timestamped logs

Per-agent completion times and per-review scores allow maintainers to localize slowness, diagnose agent regressions, and compare run quality over time.

Metrics footer

Totals, per-phase timings, plan counts, review counts, winner, runner-up, merge count, and output path make the run auditable after the fact.

Extension Points

Agent topology

Swap or expand the default agent list, but update quorum logic, scorecard formatting, and review edge generation so the matrix still matches the selected set.

Synthesis policy

Adjust dimension weights or tie handling inside the orchestrator without changing dispatcher contracts, as long as review JSON still exposes strengths, weaknesses, and averages.

Persistence behavior

Add additional history artifacts, alternate output naming, or richer PR summaries in Phase 5 while keeping --dry-run semantics intact.
