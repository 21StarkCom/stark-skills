/stark-session-insights

Analyze Claude Code session history — usage patterns, skill invocations, corrections, and preferences per project
AUDIENCE: USAGE · COMPLEXITY: MEDIUM (368 LINES)
How to Invoke
$ /stark-session-insights    # analyze all projects
$ /stark-session-insights --project infra-pulse    # single project (partial match)
$ /stark-session-insights --refresh    # force regenerate even if current
$ /stark-session-insights --project stark --refresh    # combine flags

Also triggered by: session insights analyze sessions usage patterns what do I do most

Inputs & Outputs

Input: Session History

~/.claude/history.jsonl
One JSON object per line. Fields: display (prompt text), timestamp (epoch ms), project (absolute path), sessionId.

The skill reads this file but never modifies or deletes it.

Output: Per-Project Insights

~/.claude/code-review/insights/sessions/{slug}.md
One markdown file per project. Slug derived from path: Evinced/infra-pulse → evinced--infra-pulse.

Contains all 15 analysis sections plus narrative synthesis.

Output: Summary Index

~/.claude/code-review/insights/sessions/index.md
Cross-project overview table: prompt counts, sessions, days active, date ranges, top action per project. Updated on each run.

Common Workflows

Quick Check: All Projects

Run /stark-session-insights with no flags. Processes every project in your history. Skips any project whose insights are already current (entry count matches). Takes 1–3 minutes depending on history size.

Deep Dive: Single Project

Run /stark-session-insights --project infra-pulse to focus on one repo. Partial match works — "stark" matches "Evinced/stark-skills". Useful when investigating how you used Claude on a specific effort.

Force Refresh

Run /stark-session-insights --refresh to regenerate all insight files even if they appear current. Use after history.jsonl has been edited or if a previous run was interrupted.

After a Sprint

At sprint end, run insights to see which skills you leaned on, what corrections recurred, and whether your work rhythm changed. The narrative synthesis and recommendations section turns raw data into an actionable retrospective.

Feed Into Memory

Corrections detected by the skill (Section 2.7 & 2.13) are the most valuable signals. Review them and save recurring patterns as feedback memories so Claude avoids the same mistakes in future sessions.

Spot Burnout Signals

The hourly distribution and weekly trends reveal sustainability issues. Late-night spikes, 7-day weeks, or escalating frustration markers in the correction categories are early warnings the recommendations section will flag.

Flag Reference
Flag Type Default Description
--project <name> string all projects Case-insensitive partial match on project path. Errors if no match found.
--refresh boolean false Regenerate insight files even if entry count hasn't changed since last run.
Key Constants
Constant Value Purpose
HISTORY_FILE ~/.claude/history.jsonl Source data — one JSONL entry per user prompt
OUTPUT_DIR ~/.claude/code-review/insights/sessions/ Where per-project .md files and index.md are written
SESSION_GAP 30 minutes Threshold for detecting session boundaries when sessionId is missing
Safety & Limits

Read-Only Input

Never modifies or deletes history.jsonl. All output goes to a separate directory.

Local Only

No data sent externally. No GitHub issues, PRs, or API calls. All files stay on disk.

Idempotent

Re-running with same data produces identical output. Skips current files unless --refresh.
