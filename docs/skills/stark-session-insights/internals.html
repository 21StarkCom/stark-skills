Phase 2: Analysis Modules (Inline Python)

1. Usage & Behavioral Metrics

Basic Stats: Prompt counts, date ranges, sessions length (30-min gap threshold).

Commands: Extracts tools/skills by scanning for /stark- or / prefixes.

Action Words: Analyzes keyword frequencies (review, fix, push, test, merge).

2. Temporal & Rhythm Analysis

Time Distribution: Buckets activity by local hour for ASCII bar charts.

Day-of-Week: Aggregates to determine deep work rhythms vs weekend warrior usage.

Lifecycle Trends: Maps weekly ISO volumes (ramp-up, tapering, sustained pace).

3. Session & Interaction Shapes

Prompt Sizing: Bins prompts into Short (≤50), Med (51-200), and Long (>200) categories.

Session Typing: Categorizes sessions heuristically (Build/Ship, Review, Debug, Architecture, Maintenance).

Corrections: Detects course-correction signals ("no", "don't", "stop") and groups them.

4. Automated Narrative Synthesis

Prose Generation: Weaves the metrics into a 3-5 paragraph "So What?" project story.

Recommendations: Autogenerates 3-5 prescriptive fixes tied back to exact data (e.g., workflow changes if debug sessions run long).
