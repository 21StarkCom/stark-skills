Data Model & I/O

Input: history.jsonl

Single JSONL file at ~/.claude/history.jsonl. Each line is a JSON object. Required fields: timestamp (ms epoch), project (abs path). Optional: display (prompt text, defaults to ""), sessionId (string; if missing, 30-min gap heuristic derives sessions).

Output: {slug}.md

One markdown file per project at ~/.claude/code-review/insights/sessions/{slug}.md. First line is an HTML comment with generation timestamp and entry count — this is the freshness token used by skip logic. Slug derivation: last-2-path-segments, / → --, lowercased.

Output: index.md

Cross-project summary at ~/.claude/code-review/insights/sessions/index.md. Contains a table of all projects sorted by prompt count, plus cross-project aggregates (most active project, total sessions, most used skill).

JSONL Entry Schema

{
  "display":   "string — the user's prompt text",
  "timestamp": 1742000000000,        // ms since epoch (required)
  "project":   "/Users/.../repo",    // absolute path (required)
  "sessionId": "abc123"              // optional — gap heuristic if missing
}

Freshness Token (line 1 of output .md)

<!-- generated: 2026-03-25T14:30:00, entries: 847 -->

Skip logic compares entries: N against current entry count for the project. If equal and --refresh not set → skip regeneration.

Session Detection Algorithm

Primary: sessionId field

If sessionId is present, entries are grouped directly by this value. Each unique sessionId = one session. This is the preferred path.

Fallback: 30-minute gap heuristic

When sessionId is missing: sort entries by timestamp, scan sequentially. If gap between consecutive entries exceeds SESSION_GAP (30 min = 1,800,000 ms), start a new session. Constant is defined at skill top-level — change it to adjust sensitivity.

Correction Detection Rules (2.7 & 2.13)

Detection Patterns

Prefix "no": isolated "no", "no,", "no!" — excludes "no arguments", "no changes".
Keywords: don't, stop, wrong, not what I, instead, actually, I said, I meant, undo, revert that.
Punctuation: starts with "!" or ends with "!".

Categorization (2.13)

Data/Logic: "wrong", "still not working", "broken".
UX/Direction: "don't want", "instead", "I meant".
Process/Workflow: "stop asking", "just do it", "don't write a file".
Frustration: "!" emphasis, repeated corrections.

Extension Points
1
Add a new analyzer: Insert a new section 2.N in the skill markdown between the existing analyzers and the narrative synthesis (2.14). Feed its output into the narrative and recommendations. Update the output template in Phase 3 to include a new section.
2
Change session gap threshold: Modify the SESSION_GAP = 30 constant at the top of the skill. Lower values (15 min) split more aggressively; higher values (60 min) merge interrupted work.
3
Add action words: Extend the hardcoded list in 2.3. Currently 38 words. Add domain-specific verbs (e.g., "provision", "terraform", "k8s") for infrastructure-heavy projects.
4
Add session types: Extend the classifier in 2.12. Add new categories like "Learning" (dominated by "explain", "what is", "how does") or "Migration" (dominated by "migrate", "upgrade", "convert").
5
Change output format: The Phase 3 template is pure markdown. To emit JSON or HTML instead, replace the template strings in Phase 3. The analysis engine (Phase 2) is format-agnostic.
6
Add correction patterns: Extend the regex list in 2.7. E.g., add Hebrew correction markers or team-specific shorthand. Be careful to avoid false positives — "no" is already restricted.

Failure Modes & Edge Cases

Hard Failures (Abort)

Missing history file: Errors with message and stops. No fallback.
Empty history file: Same — errors and aborts.
No project match: When --project finds nothing, prints available projects and aborts.

Soft Failures (Skip/Degrade)

Malformed JSONL lines: Skipped with warning count. Does not abort.
Missing display field: Defaults to empty string — analyzers see no text.
Missing sessionId: Falls back to 30-min gap heuristic.
Missing timestamp: Entry skipped entirely.
<2 weeks of data: Skill evolution section (2.11) omitted silently.

Degenerate Inputs

Single prompt project: Session length = 0, stats still generated.
No corrections found: Section shown with "No corrections detected."
No skills used: Section shown with "No skill invocations found."
Very long prompts: Truncated to 300 chars with "..." in output.

Safety Boundaries

Read-Only Input

Never modifies or deletes history.jsonl. Pure read-only access to the data source.

Local-Only Output

All output written to local filesystem only. No network calls, no external APIs, no uploads.

No Sensitive Analysis

Does not analyze prompt content for secrets, PII, or sensitive data. Content shown as-is (truncated).
