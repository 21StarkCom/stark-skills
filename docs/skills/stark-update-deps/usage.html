stark-update-deps

Standalone usage view for a dependency auditing and upgrade skill
Audience: usage

Update dependency manifests, verify every version, and leave a clear report.

This skill auto-discovers dependency files across a repo, researches current stable releases on official registries, applies safe upgrades in place, flags risky major bumps, and re-checks every changed version before it finishes.

What users get back

  • Updated manifest files for safe or review-worthy upgrades.
  • A decision summary: Safe, Review, Blocked, Skip.
  • Explicit verification evidence for each changed version.
  • Action items for major-version migrations and lockfile regeneration.
Critical behavior: the skill does not trust plausible version numbers. It verifies exact package versions and Docker tags against official sources before leaving them in place.

Common User Workflows

Routine Dependency Refresh

Use when you want the project brought up to current stable versions across all ecosystems in one pass. The skill auto-discovers manifests, applies safe bumps, and leaves a concise report of what changed.

Major Upgrade Triage

Use when you expect big version jumps. The skill separates safe upgrades from review-worthy ones, surfaces breaking-change risk, and points to migration guides or codemods when available.

Docker Tag Sanity Check

Use when base images or service images may be stale. The skill verifies exact image tags on Docker Hub so you do not end up with nonexistent tags that break builds later.

Guardrails And Caveats

Unverified Versions Are Not Left In Place

If an updated version cannot be confirmed after fresh searches, the skill fixes it to a confirmed version or reverts it and flags the issue for the user.

Lockfiles Are Deliberately Not Edited

The skill updates manifest intent, not generated lock state. Users should run the appropriate install command afterward to regenerate lockfiles consistently.

LTS Preference For Runtimes

For language runtimes and base images, the skill prefers stable LTS-style targets over bleeding-edge releases when ecosystem compatibility is likely to matter.
