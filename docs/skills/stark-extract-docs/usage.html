Common Patterns

Single Spec Extraction

The most common use: point at one design spec and let the skill find its associated plan and review files automatically by naming convention. ADRs go to the target repo, retrospectives stay in the source repo.

/stark-extract-docs docs/specs/2026-03-19-auth-redesign-design.md

Dry Run Preview

See what knowledge would be extracted without writing anything. Outputs the intermediate JSON showing all extractions, their categories, confidence levels, and evidence — useful for verifying before committing.

/stark-extract-docs docs/specs/my-spec-design.md --dry-run

Batch Processing

Process all design specs in a directory sequentially. Includes cross-spec ADR deduplication — if two specs produced overlapping decisions, the more detailed one is kept and the duplicate removed.

/stark-extract-docs --batch docs/specs/

Cross-Repo Extraction

When a spec in repo A describes architecture for repo B, use --target-repo to send ADRs and reference docs to B while retrospectives stay in A. The skill auto-detects org/repo references too.

/stark-extract-docs docs/specs/my-spec-design.md --target-repo ~/git/Evinced/other-repo

Force Re-Extraction

Overwrite a previous extraction run. The skill tracks what it created last time and replaces ADRs at the same numbers, swaps learning log entries, and refreshes glossary terms in place.

/stark-extract-docs docs/specs/my-spec-design.md --force

Include Low-Confidence

By default, low-confidence extractions are filtered out and medium-confidence ones get a <!-- needs review --> marker. Use --include-low to keep everything.

/stark-extract-docs docs/specs/my-spec-design.md --include-low

Artifact Discovery Convention

How the Skill Finds Related Files

Given a spec at docs/specs/2026-03-19-foo-design.md, the skill auto-discovers:

Plandocs/plans/2026-03-19-foo.md
Spec reviewdocs/specs/2026-03-19-foo-design.review.md
Plan reviewdocs/plans/2026-03-19-foo.review.md

Missing artifacts are fine — the skill extracts from whatever is available. More artifacts = richer extractions.

Edge Cases & Troubleshooting

Spec Already Processed

The skill tracks input hashes. If nothing changed since the last run, it exits cleanly. Use --force to re-extract anyway — previous outputs are overwritten at the same file paths.

Target Repo Not Found

The skill checks sibling directories and ~/git/{org}/{repo}. If neither exists, it errors with a clone suggestion. Use --target-repo <path> to point at the right location.

Dirty Target Tree

If the target repo has uncommitted changes, files are still written but the commit step is skipped with a warning. Commit your existing changes first, or use --no-commit.

No Extractable Knowledge

If the spec is too thin for any extractions, the skill exits cleanly — not an error. Consider adding more architectural detail, constraints, or a review file for richer results.

At a Glance

Two-Pass Architecture

Pass 1 extracts knowledge into structured JSON. Pass 2 routes it to the right document types and locations. Dry-run shows Pass 1 output only.

Idempotent & Safe

Hash-based skip logic prevents re-processing. Deduplication against existing ADRs, glossary terms, and log entries. Never modifies source specs.

Cross-Repo Aware

ADRs and reference docs go to the target repo. Retrospectives and learning logs stay in the source repo. Separate commits for each.
