stark-extract-docs

Extract durable knowledge from specs, plans, and review files into long-term project documentation.
Complexity: Complex (660 lines)
CLI Usage & Arguments

Standard Execution

/stark-extract-docs <path-to-spec>

Processes a single spec document. Automatically locates the associated plan and review artifacts in the same directory structure.

Batch Processing

/stark-extract-docs --batch <dir>

Iterates sequentially through all *-design.md files in the provided directory. Handles cross-spec ADR deduplication at the end.

--dry-run

Runs Pass 1 only. Displays extracted knowledge in a formatted JSON structure without writing any files or commits.

--force

Bypasses history-based skip logic. Reprocesses the spec and overwrites outputs, clearing previous learning log / glossary entries.

--no-commit

Generates and writes all documentation files locally but skips staging and committing to Git.

Notable Edge Cases & Safety

Target Repo Detection

Resolves the target repo by analyzing the spec text for org/repo strings, then searches siblings and ~/git/. If cross-repo writes are detected, retrospectives stay in the source repo while ADRs are routed to the target repo.

Confidence Filtering

Extractions marked low confidence are skipped unless --include-low is passed. medium confidence extractions are included but flagged with an inline HTML comment <!-- needs review -->.

Commit Safety

Checks git status before committing. If the target repository has a dirty working tree (uncommitted changes), it will skip the automated commit phase and instruct the user to verify and commit manually.
