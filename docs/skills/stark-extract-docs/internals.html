Artifact Resolution & Target Repo Detection

Input Artifact Convention

Given a spec at docs/{prefix}/specs/YYYY-MM-DD-foo-design.md, the skill derives 3 sibling artifacts by path convention: plan at plans/YYYY-MM-DD-foo.md, spec review at specs/...design.review.md, plan review at plans/...review.md. Each is optional — missing artifacts reduce available categories but don't block extraction.

Target Repo Resolution

Three-step resolution: (1) explicit --target-repo flag, (2) parse spec content for org/repo references (first match wins), (3) fall back to current directory's git remote. Filesystem lookup checks sibling dirs first, then ~/git/{org}/{repo}. When target ≠ source, ADRs go to target, retrospectives stay in source.

ADR Directory Detection

Checks three variants in order: docs/adr/, docs/decisions/, docs/adrs/. Uses whichever exists, defaulting to docs/adr/. Scans existing ADRs to find the highest NNNN number for auto-incrementing. Never renumbers existing ADRs.

Skip Logic & --force Behavior

Idempotency via SHA-256 Hashing

Phase 1 computes SHA-256 hashes of all input artifacts (spec, plan, reviews). On subsequent runs, these are compared against the history file's input_hashes. If all match, the run is skipped. This makes the skill safe to call repeatedly — it only re-extracts when inputs actually change.

--force Re-Run: Surgical Overwrite

When --force is passed with an existing history file, the skill reads created_artifacts from history and overwrites those specific files. ADRs keep their numbers, learning log entries are matched by slug + text and replaced, glossary terms are matched by name. Without a history file, --force simply bypasses the skip check.

Intermediate Data Format

Extraction JSON Schema (held in memory between Pass 1 → Pass 2)

schema_version: 1 • spec_path: source spec • associated_artifacts[]: found files • input_hashes{}: path → sha256 (for skip logic) • extractions[]: each has category, title, content, evidence, source, confidence (required) plus category-specific fields: has_alternatives (constraint), round (evolution), affected_agents[] (agent_signal). Validation: retry once on invalid JSON, then fail. Low-confidence items filtered unless --include-low.

Extension Points

Adding a New Category

1. Add to the 8-category table in Phase 2 with source and extraction rules. 2. Add a routing rule in Phase 3.1. 3. Implement the generator (like 3.2–3.7). 4. Add category-specific fields to extraction schema. 5. Update validation in 2.1.

Adding a New Output Type

1. Add a generation step (Phase 3.x). 2. Add dedup logic for the new file type. 3. Include in Phase 4 preview manifest. 4. Add to history file's created_artifacts for --force tracking. 5. Handle the file in --force re-run cleanup.

Cross-Tool Integration

History files at ~/.claude/code-review/history/extract-docs/ are consumed by stark-metrics for aggregation. The plan-to-tasks overlap check reads from sibling history at history/plan-to-tasks/. Both use the same {org/repo}/{slug}.json nesting convention.

Observability & Metrics

Task-Based Progress

7 TodoWrite tasks created at start, one per phase. Each set to in_progress before starting, completed when done. Batch mode adds 5-minute checkpoint logging.

Timing Protocol

T0 recorded at skill start. Per-phase durations logged. Improvement flags fire when: extraction count = 0, ADR dedup > 50% (batch), no review artifacts, or Pass 1 > 70% of total time.

History Persistence

JSON at history/extract-docs/{repo}/{slug}.json stores: input_hashes, created_artifacts, timing, extraction counts, output counts. Consumed by skip logic (this skill) and stark-metrics (aggregation).

Dual-Repo Commit Strategy

Source Repo vs Target Repo File Placement

When the spec references a different repo, outputs split across two repos. Target repo receives: ADRs, reference docs, glossary, constraints. Source repo (where spec lives) receives: retrospectives and learning log. Each gets its own commit: docs: extract knowledge from {slug} for target, docs: add review retrospective for {slug} for source. If either repo has a dirty tree, that commit is skipped with a warning — files are still written. Uses git add <specific-files>, never git add -A.
