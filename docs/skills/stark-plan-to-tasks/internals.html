Core Abstractions & Extension Points

Multi-Agent Validation Envelope

The output of Pass 2 is serialized into a breakdown.json alongside the original PLAN_CONTENT. This JSON envelope is dispatched via $SCRIPTS/plan_to_tasks_validate.py to independent LLM binaries (e.g., codex, gemini) acting as adversarial reviewers ensuring coverage and dependency sanity.

GitHub Type & Label Taxonomy

Issues are labeled dimensionally (Story Points, Risk, Confidence) but also map directly to native GitHub Issue Types (Feature, Task, Bug) using --field type="..." rather than falling back to type labels.

Dual Auth Boundary

The skill uses the stark-claude GitHub App for setup, reading, and Project GraphQL queries, but explicitly unsets GH_TOKEN during the issue creation pass to ensure the issues are correctly attributed to the human user's native CLI auth.

Automated Knowledge Retention

Before deleting the plan, Pass 4 automatically sorts unstructured architectural decisions, data models, and constraints into the repository's docs/ tree natively, preventing knowledge silos in temporary PRs or deleted markdown files.
