/stark-skill-analytics

Skill usage patterns, quality metrics, and adoption insights across all Claude Code sessions
AUDIENCE: USAGE · COMPLEXITY: MEDIUM · 369 LINES
Quick Start
$ /stark-skill-analytics
# Full report: all skills, all projects, all metrics
$ /stark-skill-analytics --skill stark-review
# Deep-dive report for a single skill
$ /stark-skill-analytics --format table
# Summary rankings table only (compact output)
What You Get

Usage Rankings

Every skill ranked by invocation count, with project spread, date range, and trend direction (rising / stable / declining / new).

Quality Signals

Timeout rates, error rates, and empty-result rates cross-referenced from run history files. Per-agent breakdowns for review skills.

Adoption Timeline

Chronological view of when each skill was first used and its growth trajectory, grouped by month.

Skill Sequences

Top 10 most common skill-to-skill sequences within sessions, revealing natural workflows.

Discovery

Find skills that are registered but never invoked, or used but unregistered. Spot dead weight and hidden gems.

Recommendations

Actionable suggestions: remove dead skills, fix high-timeout skills, promote underused winners, combine frequent sequences.

Common Workflows

Weekly Health Check

1

Run /stark-skill-analytics --format table to scan overall rankings and trends

2

Check for declining skills or high error rates in the Quality Signals section

3

Act on recommendations: remove dead skills, fix broken ones

Investigate a Skill

1

Run /stark-skill-analytics --skill stark-review

2

Review per-agent success rates and timeout breakdowns

3

Check argument patterns to understand how the skill is being used

After Shipping a New Skill

1

Wait a few days, then run the full analytics

2

Check Adoption Timeline for your skill's first appearance

3

Check Discovery section to confirm it's registered and being invoked

Optimize Workflow Sequences

1

Run full analytics and scroll to Skill Sequences section

2

Identify frequent pairs (e.g., review → improvement)

3

Consider combining common sequences into a composite skill

Error Handling

Missing history.jsonl

Fatal error. Session history is the only required input. Ensure Claude Code has been used at least once to generate this file.

No Run History

Graceful degradation. Usage metrics are still reported. Quality metrics section shows "No run history data available."

Unknown --skill Name

Reports the error and lists all skills found in history so you can pick the correct name.
