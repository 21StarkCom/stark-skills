Severity & Confidence

High Severity

Execution blocker, data loss/divergence risk, security break, or guaranteed drift. Must include concrete production failure sequence.

Medium Severity

Likely operational issue with workaround available.

Low Severity

Non-blocking improvement suggestion.

High Confidence

Verifiable from document text alone.

Medium Confidence

Requires assumption about runtime or external system behavior.

Low Confidence

Uncertain about underlying API/tool behavior. Marked UNCERTAIN.

Common Workflows

Pre-Execution Gate

Run the review before executing any migration or cutover. The closing line tells you exactly how many blockers remain. Zero blockers = safe to proceed.

Plan Iteration

Fix the High findings, then re-run the review on the updated plan. Each pass should reduce the blocker count until you reach zero.

Pair with /stark-review-plan

Use /stark-review-plan for design-level review, then /stark-review-deployment-plan for infrastructure-specific adversarial analysis.

Quality Guarantees

No False Positives

No stylistic preferences or theoretical edge cases. Every High finding includes a concrete production failure sequence.

No Fabrication

Command behavior and API semantics are never guessed. Uncertain items are explicitly marked UNCERTAIN. Inaccessible checks are DEFERRED.

Full Coverage

All 10 vectors are always evaluated. No vector can PASS by omission — concrete evidence is required.
