The Adversarial Review Workflow

Hard Constraints:

No fabricating facts.

Require concrete evidence.
Plan Ingestion
Load infrastructure or migration plan
If missing, ask user for the plan.
Pass 1: Full Read
Read the entire plan document end to end
Look for missing gates and contradictions.
Pass 2: Contradiction Scan
Target hidden assumptions and logic flaws
Pass 3: Cross-Check
Validate against repo scripts and runtime config
If inaccessible, mark as DEFERRED.
Prioritize High severity findings first if context is limited.
Pass 4: Vector Evaluation
Evaluate against 10 critical failure vectors

The 10 Required Failure Vectors

A) Partial-Failure Trap

Simulate split-brain in dual-writes. Requires explicit anti-split-brain cutover gates.

B) Imperative Idempotency

Identify imperative CI/CD commands that are not idempotent or safe for upserts.

C) Blank-Slate IaC

Assume an empty account. Flag hidden assumptions about pre-existing agents/IAM.

D) Dependency Sequencing

Validate infra and runtime dependencies. Call out phase-order breakages.

E) Reality Drift

Define owners for non-IaC resources. Require drift correction or detection mechanisms.

F) Command Validation

Validate CLI/API flags. Mark UNCERTAIN if behavior is not known.

G) Pre-Cutover Gate

Require explicit final delta sync or write freeze before a read cutover.

H) API Prerequisites

Map phases to cloud APIs. Mark FAIL if APIs are implied but undeclared.

I) Identity Lifecycle

Require explicit lifecycle for runtime identities (creation, bindings, attachment).

J) Evidence Strictness

Mark PASS by evidence only. If evidence is missing, mark FAIL or DEFERRED.

Structured Output Artifacts

The skill mandates a 5-part output response structure with no skipped sections.

1. Findings (High → Medium → Low)

Material flaws detailing: Location, Failure Scenario, Why it fails, Exact Fix, and Confidence Score.

2. Dependency Violations

Table mapping Prerequisites → Dependent Step → Breakage Risk → Fix.

3. Ownership & Drift Table

Table mapping Resources → Owner → Drift Risk → Remediation Mechanism.

4. API Prerequisite Matrix

Table mapping Capabilities → Required APIs → IaC Enablement Status.

5. Coverage Matrix

Status of all 10 vectors (PASS/FAIL/DEFERRED) with a one-line evidence summary.
