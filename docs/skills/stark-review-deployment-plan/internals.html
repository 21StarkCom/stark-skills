Failure Vector Surface
\n
\n
\n

A. Partial-Failure Trap

\n

Model both A-success/B-fail and B-success/A-fail directions. Require a single source of truth and anti-split-brain cutover gate.

\n FAIL if cutover gate is missing\n
\n
\n

B. Imperative Idempotency

\n

Flag non-idempotent create, deploy, or execute commands unless clearly upsert-safe.

\n PASS only with command-level evidence\n
\n
\n

C. Blank-Slate IaC Completeness

\n

Assume an empty account or project. APIs, service agents, IAM, and networking defaults must be explicitly provisioned.

\n FAIL on implied prerequisites\n
\n
\n

D. Dependency Sequencing

\n

Validate both infrastructure ordering and deploy/runtime order, especially secrets, IAM, and rollout-time dependencies.

\n FAIL on phase-order breakage\n
\n
\n

E. Reality Drift

\n

Every resource needs an owner and a correction mechanism, or at minimum explicit drift detection and manual remediation.

\n DEFERRED if ownership is unverifiable\n
\n
\n

F. Command Contract Validation

\n

CLI and API semantics must be validated against known behavior. Uncertainty is surfaced explicitly.

\n Use UNCERTAIN instead of guessing\n
\n
\n

G. Pre-Cutover Gate

\n

Dual-write migrations require a final delta sync or write freeze immediately before read cutover.

\n Guaranteed FAIL if absent\n
\n
\n

H. API Prerequisite Matrix

\n

Map each capability or phase to required cloud APIs and prove enablement in IaC.

\n FAIL on undeclared API dependency\n
\n
\n

I. Runtime Identity Lifecycle

\n

Track identity creation, IAM bindings, and deploy-time attachment such as --service-account.

\n FAIL on incomplete lifecycle\n
\n
\n

J. Evidence Strictness

\n

Missing evidence downgrades certainty but never permits silent PASS. Every vector needs traceable proof.

\n PASS only with concrete citation\n

Contributor Extension Points
\n
\n
\n

Cross-Check Adapters

\n

Plug in repo-specific readers for Terraform, deployment manifests, shell scripts, and runtime config so Pass 3 can convert DEFERRED checks into grounded findings.

\n
\n
\n

Cloud-Specific API Catalogs

\n

Extend the API prerequisite matrix with provider-aware capability maps so blank-slate enablement checks are precise rather than inferred.

\n
\n
\n

Command Validators

\n

Add validated command-contract knowledge for internal CLIs or custom deploy wrappers. Unknown semantics should still remain explicit as UNCERTAIN.

\n
\n
\n

Evidence Extractors

\n

Improve line-number and snippet traceability by normalizing plan sections, preserving exact quotes, and attaching resource mentions to ownership and drift rows.

\n
\n
\n

Observability Sinks

\n

Export per-pass durations, vector status counts, and blocker totals to internal telemetry or CI annotations without changing the review contract.

\n
\n
\n

Cutover Simulators

\n

Automate two-way failure simulation for dual-write migrations and generate concrete failure sequences that directly support High-severity findings.
