Operating Modes
Full Mode

Review + Fix Loop

  • Dispatches 18 sub-agents per round
  • Classifies and auto-fixes findings
  • Runs build & tests after each fix
  • Commits and pushes to PR branch
  • Loops up to N rounds (default 3)
  • Posts summary + creates bug issues

Requires: test_command in config, non-fork PR

Review-Only Mode

Review Without Fixes

  • Dispatches 18 sub-agents (one round)
  • Classifies findings
  • No code edits, no commits
  • Posts summary to PR
  • Triggered by: fork PRs, no test_command, or --dry-run

--dry-run also skips GitHub posting

What You Get

PR Comments (per agent)

Each bot (stark-claude, stark-codex, stark-gemini) posts its raw findings as a separate PR comment under its own identity. Posted automatically by multi_review.py.

Orchestrator Summary

Classified findings table with outcomes (fixed, noise, FP, ignored). Signal-to-noise ratio, misalignment analysis, and prompt improvement assessment. Posted as stark-claude[bot].

Bug Issues (critical/high)

Unresolved critical or high severity findings become GitHub issues with stark-review label. Created under your identity (not the bot). Max 5 per run.

Fix Commits

In full mode, code fixes are committed and pushed to the PR branch with messages like fix: address review findings (round 1).

History Files

Persisted to ~/.claude/code-review/history/{org}/{repo}/{pr}/: summary.md, rounds.json (with timing), and prompt-assessment.md.

Terminal Metrics

Timing breakdown per phase and round, agent success/failure counts, improvement flags (bottlenecks, high failure rate, unnecessary rounds).

Finding Classifications
Classification Counts As Criteria
fix Issue Severity ≥ fix_threshold (medium) AND actually exists in code
recurring Issue Same file ± 5 lines + same domain as a previous round's finding
false_positive Noise Described problem doesn't exist in the code
noise Noise Subjective, style preference, or contradicted by other agents
ignored Ignored Below fix_threshold (low severity) — listed for completeness

Cross-reference: 2+ agents flagging the same file + region = high_confidence

Common Workflows

Standard PR Review

Push your branch, open a PR, then run /stark-review. The skill auto-detects the PR from your current branch, reviews, fixes, and posts results.

Pre-Merge Quality Gate

Run /stark-review --rounds 1 for a single pass before merging. Quick signal on critical issues without the full fix loop.

External PR Audit

Review someone else's PR: /stark-review 42 --dry-run. Produces findings without modifying their code or posting comments.

Cross-Repo Review

Review a PR in another repo: /stark-review 15 --repo GetEvinced/other-repo. Works from any directory.
