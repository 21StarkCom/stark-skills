Usage & Configuration

How to Invoke

Use conversational triggers like "stark review", "review this PR with all agents", or the explicit CLI slash command.

/stark-review [PR_NUMBER]

Note: If PR number is omitted, the skill detects it automatically from your current branch.

Arguments

--rounds N
Max fix-review cycles (default: 3).

--dry-run
Review only. No code fixes, no PR comments.

--repo ORG/REPO
Explicitly override automatic repo detection.

Finding Classifications

Not all LLM feedback is actionable. The skill categorizes findings to filter noise:

Fix Real issues ≥ medium severity.
Recurring Found across multiple rounds.
False Positive Problem doesn't exist.
Noise Subjective style or disagreed upon by peers.
Observability & Outputs

Live Terminal Feedback

Long-running reviews provide continuous visibility:

  • Real-time task spinners mapped to phases.
  • Timestamped log lines per event.
  • 5-minute checkpoint updates for long cycles.

PR Comment Output

The orchestrator bot posts a clean summary outlining:

  • Signal-to-noise ratio (Issues vs Noise).
  • Table of fixed and recurring findings.
  • Misalignment Analysis showing root causes for false positives (missing context, prompt tuning, etc).
