Quick Start
# Simplest invocation
/stark-design-to-plan docs/specs/auth-design.md

# Pick specific agents
/stark-design-to-plan docs/specs/auth-design.md --agents claude,gemini

# Dry run (preview without writing files)
/stark-design-to-plan docs/specs/auth-design.md --dry-run

# Force run on uncommitted design file
/stark-design-to-plan docs/specs/auth-design.md --force

Common Workflows

Standard: Design Review → Plan

1. /stark-review-design docs/specs/auth-design.md
2. Address review feedback, commit
3. /stark-design-to-plan docs/specs/auth-design.md
4. Review the generated plan, then /stark-review-plan it

Quick Preview with Dry Run

Run /stark-design-to-plan spec.md --dry-run to see the scorecard and synthesis without writing files. Useful to evaluate plan quality before committing to the output.

Two-Agent Mode

For faster iteration: --agents claude,gemini uses only 2 agents (2 plans, 4 cross-reviews). Trades breadth for speed when you need a quick plan.

Typical Run Metrics

Duration

~7-10 minutes total. Phase 2 (generate) and Phase 3 (cross-review) run agents in parallel, each taking 1-3 min.

Reviews

3 plans generated, 6 cross-reviews (each plan reviewed by 2 others). 5 scoring dimensions per review.

Outputs

4 files: plan, review summary, history archive (generate.json, cross-review.json, synthesis.md, summary.md).
