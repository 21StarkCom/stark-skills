# launchd plists

Plists in this directory are **generated at install time** — they're not
checked in because they embed the absolute path of the repo checkout,
which differs per machine.

Generate them once per host with:

```bash
# from the stark-skills repo root
node --experimental-strip-types tools/observability_install_launchd.ts \
    --service all \
    --out-dir tools/observability_server/launchd \
    --log-dir "$HOME/Library/Logs/stark-observability"
```

Load the hostinfo ticker:

```bash
launchctl unload tools/observability_server/launchd/com.aryeh.observability.hostinfo.plist 2>/dev/null || true
launchctl load   tools/observability_server/launchd/com.aryeh.observability.hostinfo.plist
```

The plist sets `EnvironmentVariables.PATH` to
`/opt/homebrew/bin:/usr/local/bin:/usr/bin:/bin:...`, so `/usr/bin/env
node` resolves on both Apple Silicon Homebrew and Intel Homebrew / manual
installs.

The Phase 7 prune plist is generated by the same helper —
`com.aryeh.observability.prune.plist` runs every 3600 s (`StartInterval`)
and does not `KeepAlive`.
